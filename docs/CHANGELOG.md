*仕様書 ver 1.91 - 2026/05/09 (§5.3 描画ツールに **チャネル(平行線)** を追加。背景: ユーザーから「チャネルラインのツールを作ってほしい」との要望。**設計判断**: trendline と同じく **全 TF 表示**(平行線は時間軸上の関係を保ったまま全 TF で意味を持つ)。3 クリックで完結する作成フロー — p1 / p2 で基準線、p3 で平行線オフセット。`Drawing.kind = 'channel'`、`data = { points: [p1, p2, p3] }`(`{ t, price }` × 3)。**平行線の計算**: `slope = (p2.price - p1.price) / (p2.t - p1.t)`、`offset = p3.price - (p1.price + slope * (p3.t - p1.t))`、平行線端点 = `(p1.t, p1.price + offset)` / `(p2.t, p2.price + offset)`。`p1.t == p2.t`(垂直)の degenerate ケースは平行線・帯を描画せず基準線のみを残す。**視覚**: 基準線 + 平行線(共に色 = 作成 TF 色)+ 両線で囲まれた帯(`fillOpacity = 0.08`)。preview は phase 1(secondPoint===null)= 基準線のみ、phase 2 = 基準線 + 平行線で連続表示。**hit-test**: 3 ハンドル(p1/p2/p3 各 8px)+ 2 線分(基準線・平行線、各 6px)。drag は trendline と同じ移動戦略 — handle 個別 drag(該当点だけ更新)+ body 一括 drag(I-13 に従い `shiftPointsByPixel` で pixel anchor + xToTime snap)。実装: `drawing/tools/channel.tsx`(新規)、`drawing/state.ts` に `drawing-channel` / `moving-channel-handle` / `moving-channel-body` 状態追加(reducer は trendline 系と対称)、selectors(`cursorOf` / `previewOf` / `activeToolOf` / `startToolState`)に case 追加、`buildMovingState` の switch に channel ケース追加、`drawing/visibility.ts` の全 TF kind リストに `'channel'` 追加、`drawing/tools/registry.ts` で TOOLS に登録、`DrawingTools` の TOOL_ORDER に `channel` 追加(trendline の右隣)/ hintFor / describe 拡張、backend `schemas/drawing.py:DrawingKind` Literal / `services/session_models.Drawing.kind` コメント / `schemas/ai_analysis.DrawingSummary.kind` コメント / `frontend/api/types.ts:DrawingKind` / `spec/17-data-model.md` の例も `'channel'` 追加。テスト: `tools_hit_test.test.ts` に 9 ケース(3 ハンドル / 基準線・平行線 body / 遠方 miss / 2 点・4 点 不正データ / null API)、`state.test.ts` に 7 ケース(select-tool / 1st〜3rd click 連鎖 / mouse-move 経由の preview)を追加)*
