## 16. 段階リリース計画

### Phase 1: MVP基盤構築 (B の最小版)
- **モノレポ・uvワークスペース初期化**: ディレクトリ構成のセットアップ、`pyproject.toml` / `uv.lock` 初期化、npm workspace設定
- **shared-schema 初期版**: 最低限のテーブル定義(キャッシュテーブル含む)、alembic初期マイグレーション
- **market-data 第一版**: MT5Provider、オンデマンド取得、SQLiteキャッシュ層。MT5で主要銘柄のM5 5年分が取得可能かの検証タスクを含む(取得できない場合はDukascopy等への切り替え検討)
- **アプリB (trade-trainer) MVP**:
  - 単一時間軸チャート表示・足送り機能(market-data経由でデータ取得)
  - 買い/売り/見送り + SL/TP設定
  - 裁量決済

### Phase 2a: 分析体験の本格化
**出口条件**: 実戦レベルの分析環境でトレードできる
- `common-ui-lib` 初版(チャート、メモフォーム、描画ツール雛形)
- マルチタイムフレーム表示
- 描画ツール(水平線・トレンドライン・フィボナッチ)
- インジケーター(SMA/EMA/RSI)
- シナリオメモ機能(**最小限**: メモ本文 + エントリー根拠)

### Phase 2b: 訓練コンセプトの実装
**出口条件**: 訓練アプリ固有の価値(時間起点・スタイル・3シナリオ)が揃う
- 時間起点の銘柄選定フロー(左サイドバー銘柄一覧、詳細チャート切替、★ + 候補メモのウォッチリスト)
- **トレードスタイル管理**: スタイル定義画面、プリセット、エントリー時のスタイル選択UI、見送り時の検討スタイル記録
- シナリオメモ**拡張**(環境認識、相場観、指標認識、代替シナリオ、スタイル選定理由、銘柄選定理由、比較候補の総評)
- 波動ラベリング支援

### Phase 2c: 振り返り・事後評価・経済指標
**出口条件**: 1 セッション単位で判断の質を振り返れる / 経済指標を意識した訓練ができる
- 見送り事後検証機能(3段階の事後チャート、§9.3 OHLC ベースの機械判定ラベル表示 — **1 セッション内表示のみ、横断集計なし**)
- 後悔指標・続き観察表示(決済したトレードに対する事後チャート表示のみ、横断集計なし)
- **経済指標表示(星3・縦線+シェーディング)** + market-data に経済指標取得機能追加、CLI日次バッチ設定

### Phase 3: 運用性と習慣化 (B の完成)
- 設定画面からの market-data 経済指標手動更新ボタン
- アプリB: 途中保存・再開
- セッション履歴一覧 + 過去セッション詳細再表示(集計ではなく履歴ブラウジング、[§10](./03-analysis.md#10-集計機能を採用しない方針))
- CSV/JSONエクスポート(外部ツールで独自に集計したいユーザー向け)
- スマホUI最適化

### Phase 4: AI 分析機能 (§11)
- **§11 AI 分析** のファイル保存 + 送信プレビュー + バジェット管理
- チャート画像レンダリング、メタデータ・メモ送付、Markdown レポート表示
- **1 セッション単位のみ**(横断集計は [§10](./03-analysis.md#10-集計機能を採用しない方針) で不採用)
- 専用の集計アプリ(trade-analyzer)は持たない。AI 分析は trade-trainer 本体の機能として統合する

### Phase 5: リアルトレードアプリ・デモ版 (C の実装、デモ口座)
- **アプリC (trade-live) 第一版**:
  - common-ui-lib を利用したUI構築(配色は警告色系)
  - MT5リアルタイムデータ表示
  - MT5**デモ口座**への発注機能
  - ロット自動計算・リスク管理
  - 保有中の気づきメモ(任意、タイムスタンプ付き自由記述)
  - 発注安全機構(二段階確認のみ、発注前入力は全て任意)
  - ポジション継続管理(オープンポジション一覧、当初メモ参照、気づきメモ記録)
  - オンデマンドAI分析(trade-analyzer と連携、ユーザー明示トリガーのみ)

### Phase 6: リアルトレードアプリ・実口座 (C の本運用)
- 実口座接続(最小ロットからスタート)
- 通常ロットでの本格運用
- trade-live → trade-trainer の復習ボタン連携(リアルの負けトレードを訓練セッションとして再現)
- リアル特有の 1 セッション振り返り表示(保有時間・当初メモと決済結果の対比、横断集計は持たない)

## 17. データモデル(概要)

以下のスキーマは `shared-schema` パッケージで定義し、4つのアプリ(A/B/C/D)すべてが共有する。スキーマ変更時はパッケージを更新し、各アプリをそれに追従させる。

**Session の稼働モード**: training では時系列に1つずつ(1セッション1エントリー完結)。real では複数の Session が並行稼働する([§12.7](./04-live.md#127-ポジション継続管理) 参照)。

```
Session
├─ id, 開始日時, 提示日時(チャート内), 時間フィルタ設定
├─ mode (training | real)
├─ indicator_config_id (当時のインジケーター設定バージョン、分析時の再現用)
├─ candidates[] (候補ウォッチリスト)
│   ├─ symbol
│   ├─ memo (候補メモ: 候補を見ながら残す自由記述、§6.3.1)
│   ├─ 最終選定されたか (bool)
│   ├─ 見送り理由 (層 1、任意、§6.3.2、自由記述)
│   └─ 事後データ (10/50/200本先のOHLC。自己評価は §9.3 で OHLC から機械判定し保存しない)
├─ final_decision
│   ├─ symbol, エントリー有無
│   ├─ エントリー見送り理由 (層 2、必須、§7.3、自由記述)
│   ├─ skip_reason (全候補見送り時の自由記述、任意)
│   ├─ 検討したスタイル[] (見送り時も記録可、§8.5)
│   └─ 事後データ (§9.3 で OHLC から機械判定)
├─ 中断・再開状態

Trade (エントリー時のみ)
├─ session_id
├─ mode (training | real)
├─ エントリー日時, 方向, 価格, SL, TP
├─ 決済日時, 決済価格, 決済理由
├─ pips損益
├─ 金額損益 (realのみ)
├─ ロット (realのみ)
├─ MT5_order_id (realのみ)
├─ style_id (選択したトレードスタイル)
├─ style_selection_reason (スタイル選定理由)
└─ 後悔指標データ(続き観察期間のOHLC)

TradingStyle (ユーザー定義のトレードスタイル)
├─ id (文字列キー: 'short', 'mid', 'news' 等)
├─ name (表示名)
├─ primary_timeframe
├─ expected_hold_time
├─ expected_rr
├─ typical_sl_pips
├─ description (運用ルール・説明、自由記述)
└─ is_active (有効/無効フラグ)

Scenario (Tradeと1対1)
├─ 環境認識, 相場観
├─ 銘柄選定理由, 比較候補の総評
├─ 指標認識
├─ 波動カウント
├─ メインシナリオ, 代替シナリオ1, 代替シナリオ2
├─ エントリー根拠
├─ 決済時メモ
└─ 振り返りメモ

HoldingMemo (保有中の任意メモ、Tradeに紐づく)
├─ trade_id
├─ timestamp
└─ メモ本文(自由記述)

Drawing (Sessionに紐づく描画保存)
├─ session_id
├─ 種類 (line / fibonacci / label / trendline)
├─ 座標データ
├─ timeframe (描画を行ったチャートの時間足: M5/M15/H1/H4/D1 等)
├─ visible_on_timeframes (表示対象時間足の配列、JSON)
└─ ラベル文字

Account (realモード用、MT5から同期)
├─ 残高, 口座通貨, レバレッジ
├─ 証拠金余力
└─ 最終同期時刻

Setting
├─ 対象銘柄リスト
├─ スプレッド設定(銘柄別)
├─ 時間軸プリセット
├─ 時間フィルタプリセット
├─ 事後評価パラメータ(lookahead_bars, opportunity_pips, noise_pips、§9.3)
├─ 経済指標表示設定(重要度閾値、表示通貨、シェーディング範囲)
└─ リスク設定(1トレード許容損失%/金額)

IndicatorConfig (インジケーター設定のバージョン管理)
├─ id, 作成日時
├─ is_active (現在有効な設定か)
└─ indicators[] (種類・期間・色などのパラメータ配列)
  例: [{ type: "SMA", period: 20 }, { type: "RSI", period: 14 }]

EconomicEvent
├─ id, 発表日時
├─ 通貨, 指標名, 重要度(1-3)
├─ 実測値, 予想値, 前回値
└─ サプライズ度(計算値)
```

**DB に置かないデータ**

- **AI 分析結果**: `data/ai_analysis/{session_id}/` 配下にファイル保存([§11.7](./03-analysis.md#117-分析結果の永続化ファイルストレージ) 参照)。統計対象外・画像と長文 Markdown を含むため、DB テーブルは設けない
- **市場 OHLC**: `market-data` パッケージが管理するキャッシュ DB(SQLite)側に分離し、本スキーマでは保持しない
