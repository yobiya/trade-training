# Principle: タグ・構造化選択式入力を採用しない方針

← [仕様書インデックス](../README.md)

---

> この方針は [§6.4](../06-session-screen.md#64-通貨強弱変動率の自動表示について)、[§7.1](../07-memo.md#71-設計方針-フォーマットを押しつけない)、[§9.1](../09-post-review.md#91-2層の見送り記録)、[§11](../11-ai-analysis.md)(AI 分析)、[§18](../18-dev-priority.md)(開発優先順位)で参照される横断方針です。旧 §7.8 としてメモ仕様書の末尾に置かれていましたが、複数章から参照される方針のため本ハブに集約しています。

本仕様では、分類・集計の入り口として **タグ(固定選択式・自由ラベル)や構造化選択式入力全般** を採用しない。具体的には以下を採用対象外とする:

- シナリオ / エントリー根拠 / 見送り理由のタグ(固定語彙・自由追加)
- 事後評価の自己評価タグ(`機会損失`・`慎重すぎ` 等のユーザー選択式ラベル)
- 層 1 / 層 2 見送り時の skip_tags
- 通貨強弱の構造化入力(通貨別数値スコア・選択肢式ラベル 等)

## 理由

- **訓練の目的が言語化**であり、選択式タグは「選ぶだけ」で済ませる逃げ道になる。自由記述で毎回言語化させることを優先する
- **集計の粒度は OHLC から導出できる**(見送り後 N 本での値動き = 機会損失 / 慎重すぎ判定、保有時間帯 = セッション、指標近接 = 経済指標との時刻差)。タグで人力分類するより自動算出のほうが信頼できる([§9.3](../09-post-review.md#93-事後評価の算出方針) 参照)
- **定性パターンは AI 分析で自由記述を横断的に読む**ことで抽出する([§11](../11-ai-analysis.md))。固定語彙に縛られない分、メモの粒度がそのまま分析の粒度になる
- 「参考値」の表示や「将来欲しくなるかも」での先回り実装は、自動算出への入り口となり、自分の目で読む訓練を侵食する
