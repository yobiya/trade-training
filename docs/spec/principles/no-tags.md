# Principle: タグ・構造化選択式入力を採用しない方針

← [仕様書インデックス](../README.md)

---

> この方針は [§6.4](../06-session-screen.md#64-通貨強弱変動率の自動表示について)、[§7.1](../07-memo.md#71-設計方針-フォーマットを押しつけない)、[§9.1](../09-post-review.md#91-2層の見送り記録)、[§18](../18-dev-priority.md)(開発優先順位)で参照される横断方針です。旧 §7.8 としてメモ仕様書の末尾に置かれていましたが、複数章から参照される方針のため本ハブに集約しています。

本仕様では、分類・集計の入り口として **タグ(固定選択式・自由ラベル)や構造化選択式入力全般** を採用しない。具体的には以下を採用対象外とする:

- シナリオ / エントリー根拠 / 見送り理由のタグ(固定語彙・自由追加)
- 事後評価の自己評価タグ(`機会損失`・`慎重すぎ` 等のユーザー選択式ラベル)
- **事後 OHLC からの機械判定ラベル**(`機会損失` / `正解` / `どちらでも` 等の自動生成ラベル。閾値でユーザー判断を縮約するため、手動タグと同じ結果バイアスを生む)
- 層 1 / 層 2 見送り時の skip_tags
- 通貨強弱の構造化入力(通貨別数値スコア・選択肢式ラベル 等)

## 理由

- **訓練の目的が言語化**であり、選択式タグは「選ぶだけ」で済ませる逃げ道になる。自由記述で毎回言語化させることを優先する
- **機械判定ラベルも同じ逃げ道**: OHLC から算出したラベル(例: 20pips 以上動いたら「機会損失」)を表示すると、ユーザーが pips 数値を自分で解釈する訓練を侵食する。事後 pips 数値と事後チャート([§9.3](../09-post-review.md#93-事後評価の表示方針))はそのまま表示し、解釈はユーザーが行う
- **定性パターンはユーザー自身がメモを読み返して抽出する**。固定語彙に縛られない分、メモの粒度がそのまま振り返りの粒度になる
- 「参考値」の表示や「将来欲しくなるかも」での先回り実装は、自動算出への入り口となり、自分の目で読む訓練を侵食する
