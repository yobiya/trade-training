# Principle: 判断時点で実トレードでは知り得ない情報は訓練画面にも出さない

← [仕様書インデックス](../README.md)

---

> この方針は [§5.4](../05-chart.md#54-経済指標表示)、[§6.1](../06-session-screen.md#61-画面構成)(current_position 遮蔽)、[§9](../09-post-review.md)(判断結果の事後確認)、[§11](../11-ai-analysis.md)(AI 分析の送信データ)で参照される横断方針です。

本ツールは「1 回のトレードの精度を上げるための訓練」である以上、**判断時点で実トレードでは知り得ない情報を訓練画面に表示してはならない**。これは将来の値動きだけでなく、「実トレードの画面には通常出てこない自動要約指標」も含む。

## 採用しないもの(時点別)

### 分析中・エントリー判断時点
- **未来のローソク足**: `current_position` より右は描画しない([§6.1](../06-session-screen.md#61-画面構成))
- **未来の経済指標の実測値・サプライズ度**: 発表予定・予想値・前回値は表示するが、実測値は発表時刻到達後のみ公開([§5.4](../05-chart.md#54-経済指標表示))

### 保有中(エントリー後・決済前)
- **MFE / MAE 等の自動要約**: 保有中にリアルタイムで自動計算する取引画面は実トレードに存在しない。判断軸を「ツールが示した極値」に寄せてしまうため表示しない([§9.2](../09-post-review.md#92-事後観察データの種類))
- 現在価格・現在の含み R は表示してよい(これは実トレードの取引画面でも見える情報)

### AI 送信時
- ユーザー画面で判断時点に見られた情報と同じスコープでのみ AI にも渡す([§11](../11-ai-analysis.md))。ユーザーが見えなかった情報(未来値動き・決済後情報)は AI にも送らない

## 理由

- **訓練の目的は「判断時点で使える情報で判断する」スキルを磨くこと**。未来情報や自動要約指標を見せると判断軸が結果論にシフトし、[§11.9](../11-ai-analysis.md#119-訓練価値保全のためのプロンプト設計方針) で AI に禁止した結果バイアスと同じ構造が、ユーザー自身に発生する
- 実トレードでは MT5 のような取引画面に MFE/MAE・機会損失ラベル・「今がピークです」等の誘導は出てこない。訓練環境がそれらを提示すると、実戦復帰時に「情報がない状態」に対応できなくなる
- [principles/no-tags.md](./no-tags.md) の「結果バイアスラベルは採用しない」と、[principles/no-aggregation.md](./no-aggregation.md) の「集計を通じて成績追求に変質するのを避ける」と同じ論理軸。未来情報・自動要約・結果論ラベルは互いに代替可能な「判断軸を歪める入り口」として、同じ理由で排除する

## 許容するもの

- **決済後・見送り後の事後情報**: 実トレードでも決済後・見送り後にチャートを見返せば知り得る情報なので、訓練画面でも提示してよい(MFE / MAE / 続き観察 / 最大順行・逆行 R)
- **現在時点までの市場情報**: 現在価格・現在含み R・過去ローソク・発表済み経済指標の実測値
- **スタイル情報**: ユーザー自身が事前定義したもの(`TradingStyle.expected_rr` 等)
