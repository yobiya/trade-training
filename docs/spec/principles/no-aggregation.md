# Principle: 集計・情報蓄積を採用しない方針

← [仕様書インデックス](../README.md)

---

> この方針は [§1](../01-overview.md)(コンセプト)、[§9.4](../09-post-review.md#94-単一セッションの事後振り返り表示)、[§11.1](../11-ai-analysis.md#111-位置づけ)、[§13](../13-data-storage.md)(データ保存方針)、[§15.5](../15-tech-stack.md#155-ai-分析の配置)、[§16](../16-release-plan.md)(リリース計画)、[§18](../18-dev-priority.md)(開発優先順位)の設計の基礎になっている横断方針です。旧 §10 としてトレーナー仕様書内に置かれていましたが、複数章から参照される方針のため本ハブに集約しています。

本ツールは **「1 回のトレードの精度を上げるための訓練」** に特化する。様々な手法を毎回試して検証するための訓練環境であり、同じ手法を繰り返し実行して統計を取るものではない。そのため、**集計も情報蓄積も採用しない**。

## 採用しないもの

**集計・統計指標([§9.4](../09-post-review.md#94-単一セッションの事後振り返り表示) の個別表示を除く)**
- 勝率・プロフィットファクター・期待値・最大連勝/連敗などの基本指標
- 続き観察 / MFE / MAE などの横断集計(単一セッションでの表示は §9.5 で許容、複数セッションを跨いだ平均・比率は不採用)
- シナリオ別・時間軸別・銘柄別・スタイル別などの多次元集計
- セッション別勝率・曜日別勝率・指標絡み成績
- 見送り後の値動き統計などの横断集計(そもそも機械判定ラベル自体を採用しない — [no-tags.md](./no-tags.md))
- 連続学習日数・週次/月次レポート

**情報蓄積・履歴参照**
- 完了セッションの長期保存・一覧表示
- 過去セッションの再表示・再訪 UI
- トレード履歴データベースとしての蓄積
- CSV/JSON エクスポート(過去データの取り出し)

## 理由

- **毎回違う手法を検証する訓練スタイル**なので、過去のセッションを溜めて後から傾向を見る前提が成立しない。データを蓄積しても「同じ手法の反復」ではないためノイズに等しい
- **訓練の目的は 1 判断ごとの質を磨くこと**。集計値を追うと「成績を上げる」動機に変質して訓練価値が歪む
- **サンプル数が本質的に不足**(個人訓練では有意な統計を得るのに必要な試行数に届かない)
- **蓄積を前提にしないことで設計がシンプルになる**。UI に履歴一覧・集計画面・エクスポート機能を持たなくてよく、ユーザーは目の前の 1 セッションに集中できる

## 10.3 セッションのライフサイクル

- **進行中**: 銘柄選定 → 分析 → 判断 → エントリー/見送り → (エントリー時) 決済 → 振り返りメモ記入 → AI 分析実行
- **完了**: ユーザーが明示的にセッションを閉じる。閉じた時点でセッション関連データは破棄対象になる
- **破棄**: 完了後のセッションは UI から参照できない。物理削除のタイミングは実装判断(即時削除 or 一定期間後のバッチ削除)

**振り返りメモ後日追記への配慮**: 決済直後は興奮状態で精度が低いため「時間を置いて冷静に書く」運用が望ましい。これは **セッションを完了させず保留状態で置く** ことで実現する([§4.2](../04-session-flow.md#42-セッション管理) の途中保存・再開機能と同じ仕組み)。完了したセッションには戻れない。

## 許容される一時的な記録

セッション進行中 + 完了直後の短時間だけ、以下は保持する:
- 進行中セッションの状態(銘柄・メモ・描画・時間軸・足位置)
- 保留中セッションの継続情報
- AI 分析結果([§11.7](../11-ai-analysis.md#117-分析結果の永続化ファイルストレージ) ファイルストレージ、セッション破棄時に同時削除)

これらは「蓄積」ではなく「進行中の作業データ」として扱う。
