# Principle: 集計・情報蓄積を採用しない方針

← [仕様書インデックス](../README.md)

---

> この方針は [§1](../01-overview.md)(コンセプト)、[§9.4](../09-post-review.md#94-単一セッションの事後振り返り表示)、[§11.1](../11-ai-analysis.md#111-位置づけ)、[§13](../13-data-storage.md)(データ保存方針)、[§15.5](../15-tech-stack.md#155-ai-分析の配置)、[§16](../16-release-plan.md)(リリース計画)、[§18](../18-dev-priority.md)(開発優先順位)の設計の基礎になっている横断方針です。旧 §10 としてトレーナー仕様書内に置かれていましたが、複数章から参照される方針のため本ハブに集約しています。

本ツールは **「1 回のトレードの精度を上げるための訓練」** に特化する。様々な手法を毎回試して検証するための訓練環境であり、同じ手法を繰り返し実行して統計を取るものではない。そのため、**集計も情報蓄積も採用しない**。

## 採用しないもの

**集計・統計指標([§9.4](../09-post-review.md#94-単一セッションの事後振り返り表示) の個別表示を除く)**
- 勝率・プロフィットファクター・期待値・最大連勝/連敗などの基本指標
- 続き観察 / MFE / MAE などの横断集計(単一セッションでの表示は §9.5 で許容、複数セッションを跨いだ平均・比率は不採用)
- シナリオ別・時間軸別・銘柄別・スタイル別などの多次元集計
- セッション別勝率・曜日別勝率・指標絡み成績
- 見送り後の値動き統計などの横断集計(そもそも機械判定ラベル自体を採用しない — [no-tags.md](./no-tags.md))
- 連続学習日数・週次/月次レポート

**集計を目的とした情報蓄積**
- トレード履歴データベースとしての蓄積(集計を前提とした構造化保存)
- CSV/JSON エクスポート(過去データの取り出し → 横断集計に流れる導線)
- 過去セッションの自動分類・集約 UI

※ 単一セッションを **個別に振り返るためのファイル保持** は許容する(ver 1.45)。アプリは決着済みセッションを自動破棄せず、削除はユーザーが OS / Dropbox 上で行う。「**ファイル単位で残す ≠ 集計する**」と明確に分離する。

## 理由

- **毎回違う手法を検証する訓練スタイル**なので、過去のセッションを溜めて後から傾向を見る前提が成立しない。データを蓄積しても「同じ手法の反復」ではないためノイズに等しい
- **訓練の目的は 1 判断ごとの質を磨くこと**。集計値を追うと「成績を上げる」動機に変質して訓練価値が歪む
- **サンプル数が本質的に不足**(個人訓練では有意な統計を得るのに必要な試行数に届かない)
- **蓄積を前提にしないことで設計がシンプルになる**。UI に履歴一覧・集計画面・エクスポート機能を持たなくてよく、ユーザーは目の前の 1 セッションに集中できる

## 10.3 セッションのライフサイクル

- **進行中** (`settled_at == null`): 銘柄選定 → 分析 → 判断 → エントリー/見送り → (エントリー時) 決済。振り返りメモはまだ書かれていない
- **決着済み** (`settled_at != null`): トレード決済済み or 見送り確定済みの状態で横断メモが書かれた時点で自動遷移([§4.2.2](../04-session-flow.md#422-決着遷移条件))。後日のメモ追記・編集は引き続き可能
- **削除**: アプリ側で自動破棄しない。削除はユーザーが OS / Dropbox / エクスプローラ上でセッションディレクトリ(`data/sessions/{dir}/`)を直接削除する手動操作

旧仕様の「完了で自動破棄」「`is_suspended` による保留」は ver 1.45 で撤廃。**完了 = 編集ロックや破棄ではなく、結果が出てユーザーが振り返りに着手したマイルストーン**と再定義した。決着済みになっても note・銘柄別メモ・描画の追記は自由に行える。

## 個別保持と集計排除の境界

セッションは個別に永続保持する一方、**横断集計は持たない**。両者は両立する設計である:

| 許容(個別保持) | 不採用(集計) |
|---|---|
| 1 セッション分のファイル一式を残す | 全セッションを横断した勝率 / 期待値 / R 統計 |
| 一覧で名前・銘柄・日時・実損益 R を見る(現在の SessionListItem) | スタイル別 / 時間軸別 / 銘柄別の多次元集計 |
| 個別セッションを再度開いてメモを更新する | 連続学習日数 / 週次レポート |
| AI 分析を 1 セッション単位で何度でも実行する | 過去セッションを自動分類・要約する画面 |

「セッションが残る = 集計が始まる」ではない、という原則を守る。

## 進行中作業データの保存先

- セッションファイル(`data/sessions/{dir}/`): ユーザー入力一切([§13](../13-data-storage.md))
- AI 分析結果(`data/sessions/{dir}/ai_analysis/`、§11.7): ファイルストレージ、セッションディレクトリ削除で同時に消える
- 市場データキャッシュ(`ohlc_m5` / `economic_events` テーブル): 消耗品扱いで同期対象外
