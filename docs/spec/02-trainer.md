## 4. セッションフロー

### 4.1 通常セッション(標準フロー)

```
[Phase 1: 時間選定]
1. 時間フィルタ設定(セッション・曜日・期間など)
2. システムがランダムな日時を提示

[Phase 2: 相場観形成と銘柄選定(行き来あり)]
3. 各銘柄を詳細チャート(マルチタイムフレーム)で観察。メモパネル(§7.3、M キー or ボタン)を開閉しながら、
   銘柄別メモに個別の気づき、横断メモに通貨強弱・環境・銘柄比較を書く
4. 優位性を感じた銘柄を ★ で候補化(銘柄別メモはそのまま引き継がれる)
5. 候補を比較しながら横断メモを更新(「ドル強・円弱だから USDJPY 優位」等)
6. 1 銘柄に絞る(または全候補見送り)

[Phase 3: 選定銘柄の詳細分析とシナリオ立案]
7. 選んだペアで描画ツール・インジケーターを使って詳細分析(サポレジ・トレンドライン・波動カウント)
8. メモパネルで計画を書く:
   - 銘柄別メモ(選定銘柄): トリガー条件 / SL・TP の根拠 / 波動カウント
   - 横断メモ: シナリオ大枠(メイン / 代替)・指標観・スタイル選定の考え
9. エントリー時は §7.4 の最小項目(方向・価格・SL・TP・スタイル)を入力するだけで即確定できる状態にしておく

※ Phase 2 と Phase 3 は線形ではなく **行き来する**。詳細分析の結果「この銘柄ではない」と判断したら Phase 2 に戻って別銘柄を選び直して良い。メモも銘柄選定と Phase 3 を往復しながら書き足していく。

[Phase 4: エントリー判断]
10. 「エントリー」または「見送る」を選択(§7.6)
11. エントリー時: 方向 + 価格 + SL + TP + スタイル の最小入力のみ(§7.4)。根拠・シナリオは Phase 3 でメモに書き終えている前提
    見送り時: 確定のみ(§7.5)。検討したスタイルは任意選択

[Phase 5: 進行・決済(エントリー時のみ)]
12. ローソク足を進めて裁量決済。進行中の気づきはメモパネルで追記
13. 決済時の所感は横断メモに追記(§7.7)

[Phase 6: 振り返り]
14. 決済後の続き観察(追加50本、後悔指標)
15. 見送り事後検証(事後チャートと §9.3 自動判定指標)
16. 振り返りは横断メモに追記(後日追記可、§7.7 / §10.3)
```

### 4.2 セッション管理
- 時間制限なし
- 途中保存・再開機能あり
- **1セッション1エントリー完結**(trainer は単一判断の訓練に特化)
- 複数ポジション管理(資金配分・優先順位判断・ヘッジ・段階エントリー)は trainer の対象外。これらは実戦環境(journal)で経験として学ぶ方針
- trainer で身につけるのは「1つの局面に対するエントリー・決済判断の質」。実戦で必要な「同時に複数のポジションをどう捌くか」は journal 側で扱う
- **Phase の行き来を許容する**: §4.1 のフローは概念的な順序であり、実際の訓練ではユーザーが Phase 2(銘柄選定)と Phase 3(詳細分析)、Phase 3 と Phase 4(判断)を行き来できる。特に銘柄選定は相場観形成と不可分で、詳細分析で不適切と判断したら選定をやり直せるべき。UI は「戻る」動作を自然にサポートする(例: Phase 3 で「銘柄を選び直す」ボタンで Phase 2 に戻る)

## 5. チャート機能

### 5.1 表示

#### 5.1.1 マルチタイムフレーム縦積み
- 全 TF (M5/M15/M30/H1/H4/D1) のチャートを**画面幅フルで縦に並べる**
- 各 TF を個別に**表示/非表示切替**できる(不要な TF を隠して集中できる)
- 並び順: **エントリー足を最上段**(M5 が上、D1 が下の「エントリーから環境を俯瞰する」配置)
- 表示 TF 数に応じて各チャートが高さを等分する(減らすほど各チャートが大きくなる)
- セッションの `current_position` は全 TF 共通、足送りは全 TF 同期
- 未来部分は隠蔽、ローソク足を1本ずつ進行可能
- 上位足は形成途中の足をリアルタイム更新(下位足の進行に同期)

#### 5.1.2 時間軸・クロスヘア同期
- **クロスヘア同期**: ある TF でマウスを動かすと、他の TF 上にも同じ時刻の縦線が表示される
- **時間軸パン同期**: ある TF をスクロール・ズームすると、他の TF も対応する時間帯に追従
- 効果: 複数 TF 間の対比(D1 の戻り = H1 の押し目等)を視線移動だけで行える

#### 5.1.3 ホイール・ドラッグ操作
縦スクロールとチャートのズームが衝突するため、以下の操作体系とする。

| 操作 | 振る舞い |
| --- | --- |
| ホイール (素) | ページの**縦スクロール**(他 TF を見る) |
| Ctrl + ホイール | そのチャートの**時間軸ズーム** |
| 時間軸のドラッグ | 時間軸のスケール変更 |
| 価格軸のドラッグ | 価格軸のスケール変更 |
| チャート中央のドラッグ | 時間軸のパン |
| タッチ: ピンチ | ズーム |
| タッチ: 1 本指スワイプ | ページスクロール |

- 素のホイール = ページスクロールは Web の慣習(Ctrl + ホイール = ズームは Figma/Google Maps 等でも共通)に一致させ、学習コストを下げる
- 実装では lightweight-charts の `handleScale.mouseWheel` を無効化し、上位 DOM で `ctrlKey` 判定付きの wheel ハンドラを自前実装する

#### 5.1.4 その他
- **時刻表示は JST**: 時間軸ラベル、クロスヘア、ツールチップ全て JST 表示(内部データは UTC 保持、表示時のみ変換)
- 表示 TF の選択状態はセッションまたはユーザ設定として保存する(離脱・再入場で復元)

### 5.2 インジケーター(自前実装)
- SMA(単純移動平均)
- EMA(指数移動平均)
- RSI(サブパネル表示)
- 拡張可能な設計で実装

### 5.3 描画ツール
| ツール | 実装 | デフォルト表示範囲 |
|---|---|---|
| 水平線 | ライブラリ標準 | 全時間足 |
| トレンドライン | ライブラリ標準 | 全時間足 |
| フィボナッチリトレースメント | プラグイン自前実装 | 作成時間足のみ |

- **作成時間足の自動記録**: 描画対象となったチャートの時間足を `timeframe` として保存する。マルチタイムフレーム分析において「どの時間足を見て引いたライン/カウントか」を後から識別できる。
- **表示対象時間足の個別制御**: `visible_on_timeframes` で表示する時間足を個別指定可能。上表は初期値であり、ユーザーが描画プロパティから変更できる(例: 特定のトレンドラインを上位足でも表示したい等)。
- **時間足の視覚的識別**: 描画の色を作成時の `timeframe` に応じて固定し、一目でどの時間足を見て引いた描画かを識別できるようにする。

  | TF  | 色                         |
  | --- | -------------------------- |
  | M5  | スカイブルー (#58a6ff)      |
  | M15 | ティール (#56d4dd)          |
  | M30 | 緑 (#26a69a)                |
  | H1  | 黄 (#e3b341)                |
  | H4  | オレンジ (#f0883e)          |
  | D1  | 紫 (#d2a8ff)                |

  - ツール種別は形状で判別する(水平線・線分・フィボナッチ)、色は時間足識別に専念する
  - 描画のホバー・選択時には端点付近に TF バッジ(例: `[H1]`)を表示し、色覚配慮の補助情報とする
  - ユーザーが `visible_on_timeframes` を変更して他の時間足に表示している描画も、色・バッジ共に **作成時の時間足** を示す(どの時間足の視点で引かれたかを保持する意図)
- **波動カウント支援**: エリオット波動のカウント UI は描画ツールではなく、§16 Phase 2b "波動ラベリング支援" の専用機能として別途提供する。

### 5.4 経済指標表示
- **表示対象**: 重要度 **星3のみ**(設定で星2以上への拡張も可能な設計)
- **通貨フィルタ**: 表示中チャートの通貨ペアに関連する通貨 + USD(基軸通貨)
- **表示スタイル**:
  - **縦線マーカー**: 発表時刻に縦線(通貨別に色分け、ホバーで詳細表示)
  - **シェーディング**: 発表前後の時間帯(例: 前5分〜後30分)を背景色で塗り、ボラ上昇期間を可視化
- **時間軸上の情報公開ポリシー**(実戦で得られる情報と同じ扱い):
  - **未来の指標も表示**: 発表予定時刻、指標名、通貨、重要度、**予想値**、前回値はセッション現在時刻より未来でも表示(実戦で事前に知れる情報)
  - **実測値のみ発表時刻到達後に表示**: サプライズ度も実測値公開後に算出・表示
  - 縦線・シェーディングは未来分も含めて常時表示(次の指標までの残り時間を意識できる)
- **ホバー情報**: 指標名、通貨、発表時刻、実測値/予想値/前回値、サプライズ度(実測 vs 予想の乖離)

### 5.5 描画の保存
セッションごとに描画オブジェクトを保存し、後から視覚的に振り返り可能。

描画には以下のメタデータを保持する:
- `timeframe`: 描画を行ったチャートの時間足(必須)。判断根拠の時間軸コンテキストとしてAI分析にも活用する。
- `visible_on_timeframes`: 表示する時間足リスト(JSON 配列)。種類ごとの既定値は §5.3 参照、ユーザーが個別に変更可能。

### 5.6 技術選定
- **TradingView Lightweight Charts** + プラグイン自前実装
- React でラップ

## 6. 銘柄選定画面

### 6.1 画面構成

時間起点で提示された特定日時について、その時点で優位性が高い銘柄をユーザーが選定する画面。サムネイルだけの粗い判断では候補の優位性を正しく比較できないため、最初から詳細チャートでの精査を前提とする。

- **左サイドバー**: 銘柄リスト(縦並び) + インジケーター切替 + 選定アクション(メモ欄は常駐しない)
- **メインエリア**: トレーニング画面と同等のマルチタイムフレーム縦積みチャート([§5.1](#51-表示) と同じレイアウト。表示 TF の切替・縦スクロール・ホイール操作の挙動も同一)
- **メモはパネル切替式**(§7.3): ヘッダー右端の「📝 メモ」ボタンまたは `M` キーで銘柄別メモ + 横断メモを全画面オーバーレイで開く。`Esc` で閉じてチャートに戻る
- 描画ツールは選定中は使用しない(選定後のトレーニング画面で有効)

**この画面は「銘柄を機械的に選ぶ場」ではなく「相場観を形成しながら候補を絞る場」**。§4.1 Phase 2 の通り、詳細チャートを見ながらメモパネルで銘柄別メモ・横断メモに相場観・環境認識・比較を書き始めて、選定確定後の Phase 3 でそれらを精緻化する。

### 6.2 銘柄リスト(左サイドバー)
- §2.8 の有効ペアを縦に列挙
- 各行の表示: 銘柄名 + ★ 候補マーク のみ
- 行クリックで詳細チャートを切替。現在表示中の銘柄はハイライト
- **直近変動率・通貨強弱などの要約指標は表示しない**。ユーザーが詳細チャートを見て自分の目で判断する(§6.4)

### 6.3 候補管理(ウォッチリスト)

複数候補を比較してから絞る訓練のため、★ マーキング + 候補メモ方式を採用する。

#### 6.3.1 候補の追加と候補メモ
- 「★ 候補に追加 / 解除」ボタンで現在表示中の銘柄を比較候補として印付け
- 各候補には**候補メモ**(自由記述)を残せる
  - 例: USDJPY に「ドル強度明確」、EURJPY に「方向感は良いが M5 で決め手なし」
  - 候補を見ている最中に書いた生の判断をそのまま残す(後からまとめて書くと記憶が曖昧になる)
  - 選定確定時、ウォッチリストに残った非選定銘柄のメモは **層 1 見送り理由** に引き継がれる(編集可)

#### 6.3.2 選定確定と層 1 見送り記録
「この銘柄で選定 →」押下時:
- ウォッチリストに**他の候補が残っている場合**、確認フォームが表示される
- 各非選定銘柄について「見送り理由(自由記述、任意)」を確認・編集できる
- 候補メモが記入済みならそれが初期値として入る(編集して確定可)
- 記入は**任意**(訓練価値を重視するユーザーは書く。素早く選定したい場合はそのまま確定可)
- 確定で:
  - 選定銘柄 = `Session.final_decision.symbol`
  - 非選定銘柄 = `Session.candidates[]` に `見送り理由` 付きで永続化(§9.1 層 1)

#### 6.3.3 全候補見送り
- 「全候補を見送り」で session 全体を skip
- 理由の記入は**任意**(優位性無しで即スキップのケースが多い)
- 任意で自由記述を残せる(`Session.final_decision.skip_reason`)

#### 6.3.4 DB 永続化
- ウォッチリストへの ★ 追加 / メモは選定確定まで client 側保持、確定時に `Session.candidates[]` へ一括保存
- 中断・再開時に失われないよう、任意のタイミングで永続化する設計も検討対象(実装時に判断)

### 6.4 通貨強弱・変動率の自動表示について

**自動算出・ダッシュボード・構造化入力・銘柄別の直近変動率表示は実装しない**。ユーザーが生チャートから自分の目で強弱を判断する訓練方針を貫く。

- 直近変動率(% 表示)は「最近動いた」という事実しか伝えず、これから優位性があるかの情報にならない。目に入るとその銘柄へのバイアスが生じるため、サイドバーには表示しない

- 読みは横断メモ([§7.2.2](#722-横断メモ))に**自由記述**で記録する
- 構造化入力(通貨別数値・選択式タグ等)は不採用([§7.8 タグ・構造化選択式入力を採用しない方針](#78-タグ構造化選択式入力を採用しない方針) 参照)
- 横断メモは 1 セッション単位の AI 観察([§11](./03-analysis.md#11-ai分析機能))の視点セクションの対象になる。横断集計は採用しない([§10](./03-analysis.md#10-集計情報蓄積を採用しない方針))

## 7. メモ機能(自由記述ベース、2 本立て)

以下はすべて**トレーニング(trainer)モード専用**。リアル(journal)モードでは発注前入力に必須項目を設けない([§12.5](./04-live.md#125-発注前入力の任意性と安全機構) 参照)。

### 7.1 設計方針: フォーマットを押しつけない

**集計を行わない方針([§10](./03-analysis.md#10-集計情報蓄積を採用しない方針))** と整合して、メモ入力もフィールド分けやフォーマットを強制しない。

- 「環境認識」「相場観」「指標認識」「銘柄選定理由」「メインシナリオ」「代替シナリオ」「エントリー根拠」「スタイル選定理由」などの項目を **別々の必須フィールドに分けない**
- 訓練の目的は「思考プロセスを言語化すること」。項目分けは書く内容のヒントになり得る一方、必須化すると「埋めるための作文」になって言語化の質が落ちる
- ただし実運用では **銘柄固有の観察と銘柄横断の観察が明確に別のもの** として生じる(例: 「USDJPY の H4 チャート形状」と「ドル全面高で JPY クロスが優位」)。この 2 つを 1 本のテキストに混在させると読み返しにくいため、メモを **2 本立て** にする

**Markdown 見出しテンプレートの扱い**: 書く内容のヒントとして、空のメモに見出しテンプレート(例: `## 相場観` `## シナリオ` 等)をあらかじめ初期挿入することは許容する([§7.2.3](#723-見出しテンプレート初期値))。これは「フィールド分け・必須化」とは異なり、**見出し自体はユーザーが自由に削除・編集・並び替えできる単なるテキスト初期値** であるため、本節の「フォーマット強制しない」方針と矛盾しない。見出しが不要なら全削除して空白からスタートしてよい。

### 7.2 メモの種類(2 本立て)

| 種類 | 紐付き | 用途 | データモデル |
|---|---|---|---|
| **銘柄別メモ**(§7.2.1) | 各候補銘柄 | その銘柄のチャート観察・トレンド・根拠・個別判断 | `SessionCandidate.memo`(既存) |
| **横断メモ**(§7.2.2) | セッション全体 | 通貨強弱・銘柄比較・相場観・シナリオ・見送り理由・決済所感・振り返り | `Session.note` TEXT(既存) |

判断に迷う書き分けはしない:
- 「この銘柄」の観察は銘柄別メモ
- 「銘柄横断・セッション全体」の観察は横断メモ
- どちらか曖昧なら横断メモに書いて問題ない

#### 7.2.1 銘柄別メモ

- 銘柄選定画面の候補 ★ 追加時から書き始められる(既存の候補メモ、§6.3.1)
- 候補化された時点で見出しテンプレート([§7.2.3](#723-見出しテンプレート初期値))が初期挿入される
- 選定確定後のトレーニング画面でも編集継続可能(選定銘柄の観察を追記する)
- 選定確定時に非選定候補のメモは **層 1 見送り理由** に引き継がれる(編集可、§6.3.2)
- 保有中の気づきメモ([§12.6](./04-live.md#126-心理状態の記録))は journal(リアル)固有なので別管理

#### 7.2.2 横断メモ

- セッション 1 つにつき 1 つの自由記述テキスト
- セッション開始から完了(§10.3 破棄)まで一貫して編集可能
- 空の状態で新規セッションを開始すると、見出しテンプレート([§7.2.3](#723-見出しテンプレート初期値))が初期挿入される。不要なら削除してよい
- 1 セッション単位の AI 分析([§11](./03-analysis.md#11-ai分析機能))に全文送付される

#### 7.2.3 見出しテンプレート(初期値)

空のメモを開いた時の初期テキストとして、Markdown 見出しテンプレートをあらかじめ挿入する。書く内容のヒントとして機能する一方、**必須フィールドではない**: ユーザーは自由に削除・追加・並び替えできる。

**銘柄別メモの既定テンプレート:**

```
## チャート観察(TF・トレンド・形状)

## サポレジ・水準

## 根拠 / 狙いどころ

## SL / TP の考え
```

**横断メモの既定テンプレート:**

```
## 相場観・通貨強弱

## 銘柄比較

## シナリオ(メイン / 代替)

## スタイル選定

## 判断

## 決済・振り返り
```

**運用ルール:**

- テンプレートは **新規セッション開始時 / 新規候補追加時** にのみ挿入される。既に記入があるメモには再挿入しない(既存内容を消さない)
- 設定画面(§Setting 相当)でユーザーがテンプレート内容を編集可能。編集した内容は以降の新規作成に反映される
- テンプレート機能自体を無効化する設定も用意する(無効化時は空文字列でスタート)
- 見出し未使用のセクションは削除推奨(空の見出しが残っていると AI 分析時のノイズになる)

### 7.3 UI: メモパネル切替式

チャートとメモを同時表示するとチャートが狭くなるため、**メモパネルはサイドバーに常駐させず、ワンアクションで開閉する** 方式を採用する。

#### 7.3.1 切替トリガー

| 方法 | 動作 |
|---|---|
| ヘッダー右端の「📝 メモ」ボタン | クリックでメモパネルをトグル表示 |
| キーボードショートカット `M` | メモパネル開閉トグル |
| `Esc` キー | メモパネル表示中に押すと閉じる |

#### 7.3.2 メモパネルの構造

チャート領域全体を覆うパネル(モーダル風オーバーレイ)。縦に 2 段:

```
┌─────────────────────────────────────────────┐
│ 📝 メモ             [× 閉じる / Esc]        │
├─────────────────────────────────────────────┤
│ 銘柄別メモ: [USDJPY ▼]                      │
│ ┌─────────────────────────────────────────┐ │
│ │ ドル強度明確、H1 でダブルトップ形成中   │ │
│ └─────────────────────────────────────────┘ │
├─────────────────────────────────────────────┤
│ 横断メモ(相場観・比較)                    │
│ ┌─────────────────────────────────────────┐ │
│ │ ドル全面高、クロス円は円売り混じりで    │ │
│ │ USDJPY が優位。EURJPY は方向感弱い      │ │
│ └─────────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
```

- 上段: **銘柄別メモ**。ドロップダウンで対象銘柄を切替可
- 下段: **横断メモ**(セッション全体で 1 つ)
- 両方とも自由記述 textarea。編集は即時保存(フォーカス離脱 or 一定間隔)

#### 7.3.3 画面別の銘柄別メモ対象

| 画面 | 上段(銘柄別メモ)の対象 |
|---|---|
| 銘柄選定画面 | 現在チャート表示中の銘柄が初期選択。候補以外でも表示中の銘柄のメモに切り替わる |
| トレーニング画面 | 選定済み銘柄を既定表示。ドロップダウンから層 1 候補のメモも閲覧可(層 1 は編集不可、振り返り用) |

### 7.4 エントリー時の必須項目(最小限)

「ボタンを押したら即発注できる」状態にするための数値情報のみ必須。思考・根拠はメモに書く(銘柄別または横断)。

| 項目 | 必須 | 内容 |
|---|---|---|
| 方向 | ✓ | BUY / SELL |
| 価格 | ✓ | エントリー価格 |
| SL | ✓ | ストップロス価格 |
| TP |  | 利確価格(任意) |
| スタイル | ✓ | 定義済みスタイルから選択([§8](#8-トレードスタイル管理)) |

※ スタイル選定理由・エントリー根拠・シナリオ等はメモに自由記述。フォームに独立フィールドは設けない。

### 7.5 見送り時の必須項目

なし。「検討したスタイル」を記録したい場合はスタイル一覧から任意選択できる軽量 UI を提供する([§8.5](#85-見送り時のスタイル記録))。

### 7.6 判断フロー(UI 流れ)

1. **分析段階**: メモパネル(M キー or ボタン)で銘柄別メモ・横断メモを書きながら、閉じてチャートを精査する往復
2. **判断アクション**: 下部バーの「エントリーする」または「見送る」を選択
   - エントリー: 方向 / 価格 / SL / TP / スタイル の数値入力だけ。確定で発注記録
   - 見送り: 確定で見送り記録
3. **保有中 / 決済後**: メモパネルを開いて気づき・決済所感を追記
4. **振り返り**: 横断メモに時間を置いて追記可能。決済直後は興奮状態で精度が低いため、冷静になってから書くのを推奨

### 7.7 決済・振り返りの扱い

- **決済理由**(TP 到達 / SL 到達 / 裁量判断)は Trade レコードに数値として残す(§9.3 機械判定で参照)。言葉での補足は横断メモに書く
- **振り返り**(実展開とシナリオの対応、改善点、体調・心理・判断の質への気づき)は横断メモに追記。独立フィールドは設けない
- **後日追記**: セッションを完了させず保留状態で置くことで実現([§10.3](./03-analysis.md#103-セッションのライフサイクル))

### 7.8 タグ・構造化選択式入力を採用しない方針

本仕様では、分類・集計の入り口として **タグ(固定選択式・自由ラベル)や構造化選択式入力全般** を採用しない。具体的には以下を採用対象外とする:

- シナリオ / エントリー根拠 / 見送り理由のタグ(固定語彙・自由追加)
- 事後評価の自己評価タグ(`機会損失`・`慎重すぎ` 等のユーザー選択式ラベル)
- 層 1 / 層 2 見送り時の skip_tags
- 通貨強弱の構造化入力(通貨別数値スコア・選択肢式ラベル 等)

理由:

- **訓練の目的が言語化**であり、選択式タグは「選ぶだけ」で済ませる逃げ道になる。自由記述で毎回言語化させることを優先する
- **集計の粒度は OHLC から導出できる**(見送り後 N 本での値動き = 機会損失 / 慎重すぎ判定、保有時間帯 = セッション、指標近接 = 経済指標との時刻差)。タグで人力分類するより自動算出のほうが信頼できる([§9.3](./03-analysis.md#93-事後評価の算出方針) 参照)
- **定性パターンは AI 分析で自由記述を横断的に読む**ことで抽出する([§11](./03-analysis.md#11-ai分析機能))。固定語彙に縛られない分、メモの粒度がそのまま分析の粒度になる
- 「参考値」の表示や「将来欲しくなるかも」での先回り実装は、自動算出への入り口となり、自分の目で読む訓練を侵食する

## 8. トレードスタイル管理

状況に応じて複数のトレードスタイルを使い分ける訓練のため、スタイルを構造化して管理する。

### 8.1 スタイルの考え方
- 短期・中期・指標狙いなど、トレードの性格が異なる複数のスタイルを定義
- セッション開始時ではなく**分析後・エントリー前に選択**(状況判断の訓練として)
- スタイル選択もシナリオメモに記録し、1 セッション単位の AI 分析([§11](./03-analysis.md#11-ai分析機能))でスタイル選定と実運用のズレを観察対象にする(横断集計は持たない、[§10](./03-analysis.md#10-集計情報蓄積を採用しない方針))

### 8.2 スタイル定義(ユーザー定義可能)
ユーザーが設定画面から追加・編集・無効化可能。

**スタイル定義項目**

| 項目 | 内容 | 例 |
|---|---|---|
| id | 識別子 | `short`, `mid`, `news` |
| name | 表示名 | 短期トレード、中期トレード、指標トレード |
| primary_timeframe | 主要時間軸 | M5, H1 |
| expected_hold_time | 想定保有時間 | 10分〜1時間、数時間〜1日 |
| expected_rr | 想定リスクリワード | 1:1.5, 1:2 |
| typical_sl_pips | 典型的なSL幅 | 10〜20pips, 30〜50pips |
| description | 運用ルール・説明 | 自由記述 |
| is_active | 有効/無効フラグ | true/false |

### 8.3 初期プリセット(例)
新規ユーザー向けに以下をプリセットとして提供。ユーザーが自由に編集・削除可能。

- **短期トレード** (`short`): M5基準、10分〜1時間保有、RR1:1.5、SL10〜20pips
- **中期トレード** (`mid`): H1基準、数時間〜1日保有、RR1:2、SL30〜50pips
- **指標トレード** (`news`): 強指標発表前後、M5/M15基準、特殊戦略
- **スイング** (`swing`): H4/D1基準、数日保有(任意)

### 8.4 スタイル選択UI
- エントリー前のシナリオメモ入力画面で**スタイルをドロップダウン選択**
- 選択したスタイルの想定SL幅・RRが自動表示され、SL/TP設定の参考になる
- ただし自動入力はせず、ユーザー自身がSL/TPを入力(想定と実際のズレを計測するため)

### 8.5 見送り時のスタイル記録
見送りを選んだ場合でも、**検討したスタイル**を記録できる。
- 「短期で入ろうとしたが条件合わず見送り」
- 「中期視点では入れるが短期ではダメ」等の判断を残す
