## 4. セッションフロー

### 4.1 通常セッション(標準フロー)

```
[Phase 1: 時間選定]
1. 時間フィルタ設定(セッション・曜日・期間など)
2. システムがランダムな日時を提示

[Phase 2: 銘柄選定]
3. 対象ペア一覧画面(サムネイルチャート表示)
4. 複数候補をウォッチリストに追加
5. 候補を詳細チャートで比較
6. 1つに絞る(または全候補を見送り)

[Phase 3: 分析]
7. 選んだペアのマルチタイムフレームチャート表示
8. 描画ツール・インジケーターで分析

[Phase 4: シナリオ入力]
9. シナリオメモ入力(必須項目あり)
10. 3シナリオ事前宣言

[Phase 5: エントリー判断]
11. 買い/売り/見送り選択
12. SL/TP設定(エントリー時)

[Phase 6: 進行・決済]
13. ローソク足を進めて裁量決済
14. 決済時メモ入力

[Phase 7: 振り返り]
15. 決済後の続き観察(追加50本)
16. 後悔指標表示
17. 見送り事後検証
18. 自己評価タグ付け
19. 総合振り返りメモ
```

### 4.2 セッション管理
- 時間制限なし
- 途中保存・再開機能あり
- **1セッション1エントリー完結**(trainer は単一判断の訓練に特化)
- 複数ポジション管理(資金配分・優先順位判断・ヘッジ・段階エントリー)は trainer の対象外。これらは実戦環境(journal)で経験として学ぶ方針
- trainer で身につけるのは「1つの局面に対するエントリー・決済判断の質」。実戦で必要な「同時に複数のポジションをどう捌くか」は journal 側で扱う

## 5. チャート機能

### 5.1 表示
- **マルチタイムフレーム同時表示**(メイン時間軸 + 上位足2〜3枚)
- 未来部分は隠蔽、ローソク足を1本ずつ進行可能
- 上位足は形成途中の足をリアルタイム更新(メイン足の進行に同期)
- 時間軸セットはプリセット可能(例: M15メイン + H1/H4/D1)
- **時刻表示はJST**: 時間軸ラベル、クロスヘア、ツールチップ全てJST表示(内部データはUTC保持、表示時のみ変換)

### 5.2 インジケーター(自前実装)
- SMA(単純移動平均)
- EMA(指数移動平均)
- RSI(サブパネル表示)
- 拡張可能な設計で実装

### 5.3 描画ツール
| ツール | 実装 | デフォルト表示範囲 |
|---|---|---|
| 水平線 | ライブラリ標準 | 全時間足 |
| トレンドライン | ライブラリ標準 | 全時間足 |
| フィボナッチリトレースメント | プラグイン自前実装 | 作成時間足のみ |
| テキストラベル(波動カウント用) | プラグイン自前実装、連続クリックで自動採番支援 | 作成時間足のみ |

- **作成時間足の自動記録**: 描画時のメイン時間足を `timeframe` として保存する。マルチタイムフレーム分析において「どの時間足を見て引いたライン/カウントか」を後から識別できる。
- **表示対象時間足の個別制御**: `visible_on_timeframes` で表示する時間足を個別指定可能。上表は初期値であり、ユーザーが描画プロパティから変更できる(例: 特定のトレンドラインを上位足でも表示したい等)。

### 5.4 経済指標表示
- **表示対象**: 重要度 **星3のみ**(設定で星2以上への拡張も可能な設計)
- **通貨フィルタ**: 表示中チャートの通貨ペアに関連する通貨 + USD(基軸通貨)
- **表示スタイル**:
  - **縦線マーカー**: 発表時刻に縦線(通貨別に色分け、ホバーで詳細表示)
  - **シェーディング**: 発表前後の時間帯(例: 前5分〜後30分)を背景色で塗り、ボラ上昇期間を可視化
- **時間軸上の情報公開ポリシー**(実戦で得られる情報と同じ扱い):
  - **未来の指標も表示**: 発表予定時刻、指標名、通貨、重要度、**予想値**、前回値はセッション現在時刻より未来でも表示(実戦で事前に知れる情報)
  - **実測値のみ発表時刻到達後に表示**: サプライズ度も実測値公開後に算出・表示
  - 縦線・シェーディングは未来分も含めて常時表示(次の指標までの残り時間を意識できる)
- **ホバー情報**: 指標名、通貨、発表時刻、実測値/予想値/前回値、サプライズ度(実測 vs 予想の乖離)

### 5.5 描画の保存
セッションごとに描画オブジェクトを保存し、後から視覚的に振り返り可能。

描画には以下のメタデータを保持する:
- `timeframe`: 作成時のメイン時間足(必須)。判断根拠の時間軸コンテキストとしてAI分析にも活用する。
- `visible_on_timeframes`: 表示する時間足リスト(JSON 配列)。種類ごとの既定値は §5.3 参照、ユーザーが個別に変更可能。

### 5.6 技術選定
- **TradingView Lightweight Charts** + プラグイン自前実装
- React でラップ

## 6. 銘柄選定画面

### 6.1 サムネイル一覧
- 対象ペア全ての**ミニチャート**をグリッド表示
- 各ペアに複数時間軸(例: H4 + D1)のミニチャートを並列
- 直近変動率を数値表示
- 時間軸切り替え可能

### 6.2 候補管理
- ペアクリックで「候補に追加」
- 複数候補を**ウォッチリスト化**
- 候補同士を詳細チャートで比較
- 最終的に1つ選定 or 全候補見送り

### 6.3 通貨強弱について

**自動算出・ダッシュボード・構造化入力は実装しない**。ユーザーが生チャートから自分の目で強弱を判断する訓練方針を貫く。

- 読みは相場観メモ(§7.1)に**自由記述**で記録する
- 構造化入力(通貨別数値・選択式タグ等)を意図的に採用しない理由:
  - 選択肢から選ぶと判断が選択肢に引きずられ、自分の目で読む訓練が薄れる
  - 「参考値」の表示は自動算出への入り口となり、訓練方針を侵食する
  - 「将来欲しくなるかも」で先回り実装すると本来の訓練価値を損なう
- 集計・分析が必要になった場合は、自由記述を AI 分析([§11](./03-analysis.md#11-ai分析機能))のテキストマイニングで抽出する

## 7. シナリオメモ機能

### 7.1 エントリー時入力項目

※以下の必須項目は**トレーニング(trainer)モード専用**。リアル(journal)モードでは発注前入力に必須項目を設けない([§12.5](./04-live.md#125-発注前入力の任意性と安全機構) 参照)。

| 項目 | 必須 | 内容 |
|---|---|---|
| 環境認識 | ✓ | 上位足のトレンド、相場状況 |
| 相場観 | ✓ | 自分で読んだ通貨強弱 |
| 銘柄選定理由 | ✓ | なぜこのペアを選んだか |
| 見送った候補と理由 | ✓ | 他の候補と選ばなかった理由 |
| 指標認識 | ✓ | 近くの強指標の有無と影響評価(「直近なし」も選択肢) |
| **選択スタイル** | ✓ | 定義済みスタイルから選択([§8](#8-トレードスタイル管理) 参照) |
| **スタイル選定理由** | ✓ | なぜこのスタイルを選んだか、他スタイルを選ばなかった理由 |
| 波動カウント | | エリオット仮説 |
| メインシナリオ | ✓ | 想定する主要な展開 |
| 代替シナリオ1 | ✓ | 起こりうる展開パターン2 |
| 代替シナリオ2 |  | 起こりうる展開パターン3(思いつかない場合は空欄可) |
| エントリー根拠 | ✓ | 具体的なトリガー |
| タグ | ✓ | 固定/自由タグから選択 |

**代替シナリオ2を任意とする理由**: 無理に3つ目を捻り出すと記入の質が落ちる一方、「2つしか思い浮かばない」という事実自体が**視野狭窄の可能性を示すシグナル**として学習価値を持つ。空欄を許容し、代替シナリオ2の記入有無・記入率を後から集計・AI分析の観点に加えることで、思考の多様性そのものを評価対象とする。

### 7.2 決済時入力項目
- 決済理由(TP到達 / SL到達 / 裁量判断)
- 裁量決済の場合: なぜ切ったか

### 7.3 振り返りメモ(任意・後日追記可)
- 記入したシナリオのうち実際の展開がどれに最も近かったか
- 改善点・次回への教訓
- **体調・心理・判断の質**に関する気づき(影響があったと自覚した場合のみ、自由記述。例: 「寝不足で集中できず早決済した」「連敗後で焦っていた」)
- 決済から時間を置き、冷静になってから記入するのを推奨

### 7.4 タグ機能
- **固定タグ候補**: `#押し目買い` `#戻り売り` `#ブレイクアウト` `#レンジ逆張り` `#3波狙い` `#C波狙い` `#ダマシ警戒` `#指標前` `#指標後` `#指標スキップ` `#指標無風` など
- **自由タグ**: ユーザー追加可能

## 8. トレードスタイル管理

状況に応じて複数のトレードスタイルを使い分ける訓練のため、スタイルを構造化して管理する。

### 8.1 スタイルの考え方
- 短期・中期・指標狙いなど、トレードの性格が異なる複数のスタイルを定義
- セッション開始時ではなく**分析後・エントリー前に選択**(状況判断の訓練として)
- スタイル選択もシナリオメモに記録し、後から集計・分析

### 8.2 スタイル定義(ユーザー定義可能)
ユーザーが設定画面から追加・編集・無効化可能。

**スタイル定義項目**

| 項目 | 内容 | 例 |
|---|---|---|
| id | 識別子 | `short`, `mid`, `news` |
| name | 表示名 | 短期トレード、中期トレード、指標トレード |
| primary_timeframe | 主要時間軸 | M5, H1 |
| expected_hold_time | 想定保有時間 | 10分〜1時間、数時間〜1日 |
| expected_rr | 想定リスクリワード | 1:1.5, 1:2 |
| typical_sl_pips | 典型的なSL幅 | 10〜20pips, 30〜50pips |
| description | 運用ルール・説明 | 自由記述 |
| is_active | 有効/無効フラグ | true/false |

### 8.3 初期プリセット(例)
新規ユーザー向けに以下をプリセットとして提供。ユーザーが自由に編集・削除可能。

- **短期トレード** (`short`): M5基準、10分〜1時間保有、RR1:1.5、SL10〜20pips
- **中期トレード** (`mid`): H1基準、数時間〜1日保有、RR1:2、SL30〜50pips
- **指標トレード** (`news`): 強指標発表前後、M5/M15基準、特殊戦略
- **スイング** (`swing`): H4/D1基準、数日保有(任意)

### 8.4 スタイル選択UI
- エントリー前のシナリオメモ入力画面で**スタイルをドロップダウン選択**
- 選択したスタイルの想定SL幅・RRが自動表示され、SL/TP設定の参考になる
- ただし自動入力はせず、ユーザー自身がSL/TPを入力(想定と実際のズレを計測するため)

### 8.5 見送り時のスタイル記録
見送りを選んだ場合でも、**検討したスタイル**を記録できる。
- 「短期で入ろうとしたが条件合わず見送り」
- 「中期視点では入れるが短期ではダメ」等の判断を残す
