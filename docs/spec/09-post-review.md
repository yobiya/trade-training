# §9. 見送り検証機能

← [仕様書インデックス](./README.md)

---

## 9.1 2層の見送り記録

**層1: 非エントリー候補** - 分析中に ★ でウォッチしたがエントリーしなかった銘柄
- 入力タイミング: なし(★ でメモを書いた時点でその銘柄別メモが記録。エントリーアクションで別銘柄が確定した時点で自動的に層 1 扱いになる)
- 保存先: `Session.candidates[].memo`(銘柄別メモ、自由記述)
- 見送り理由の独立フィールドは設けない。書きたければ銘柄別メモに自由記述([§6.3.2](./06-session-screen.md#632-エントリーと非エントリー候補))

**層2: セッション見送り** - そのセッションではどの銘柄にも入らないと判断
- 入力タイミング: セッション画面の「見送り」アクション([§6.3.3](./06-session-screen.md#633-セッション見送り))
- 保存先: `Session.final_decision.skip_reason`(自由記述、任意) / `considered_styles`(検討したスタイル、任意)
- 任意入力(素早く見送りたいユーザを妨げない)

## 9.2 事後データ表示
セッション終了時、見送ったペアについて以下を表示:
- **3段階の事後チャート**(10本先 / 50本先 / 200本先)
- 各期間の最大上昇幅・最大下落幅
- 買い/売り想定時の推定成績
- 実際にエントリーしたペアとの比較

## 9.3 事後評価の算出方針

自己評価タグは持たない([principles/no-tags.md](./principles/no-tags.md))。セッション時刻を起点にした事後 OHLC から正解/機会損失を機械的に判定する。

**共通パラメータ(設定で調整可)**

- `lookahead_bars`: 事後評価に使う本数(初期値 50 本)
- `opportunity_pips`: この pips 以上動いたら「機会損失」と見なす閾値(銘柄別、初期値は JPY クロス 20pips / ドルストレート 20pips 相当)
- `noise_pips`: この pips 以下の動きは「どちらでもない」と見なす閾値(初期値 10pips)

**層 1(非エントリー候補)判定**

- `機会損失`: ★ で候補化したが入らなかった銘柄について、presented_at 起点の `lookahead_bars` 以内に `opportunity_pips` 以上動いた(絶対値ベース)
- `正解`: `lookahead_bars` 内の最大変動が `noise_pips` 以下
- `どちらでも`: 上記のどちらにも該当せず(中間帯)
- 想定方向は保存しないため、判定は絶対値(最大上昇・最大下落の大きい方)で行う

**層 2(セッション見送り)判定**

- `機会損失_エントリー`: 見送り後 `lookahead_bars` 以内に、いずれかの対象銘柄が `opportunity_pips` 以上動いた
- `正解_ノーエントリー`: 対象銘柄のいずれも最大変動が `noise_pips` 以下
- `どちらでも`: 上記のどちらにも該当しない中間帯

想定方向を持たないため、判定は絶対値ベースで行う。自由記述の「見送り理由」から方向性を推定するのは §11 AI 分析の役目。

## 9.4 単一セッションの事後振り返り表示

§9.3 の機械判定は **1 セッション単位の振り返り表示** にのみ使う。横断集計(見送り正解率・機会損失率・慎重すぎ率 等の分母を取って平均する指標)は提供しない([principles/no-aggregation.md](./principles/no-aggregation.md) 参照)。

セッション終了後の振り返り画面では、そのセッションに限って以下を表示する:
- 見送った候補ごとの判定ラベル(`機会損失` / `正解` / `どちらでも` 等)と最大絶対変動 pips
- 事後 OHLC を用いた 10/50/200 本先チャート(§9.2)
- エントリー済みの場合: `pips_pnl`、SL/TP 到達位置

数値はその場で確認して自分で解釈し、必要なら次のセッションの判断材料にする。複数セッションを横断した勝率・期待値・傾向分析は AI・ローカル共に持たない。
