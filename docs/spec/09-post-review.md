# §9. 見送り検証機能

← [仕様書インデックス](./README.md)

---

## 9.1 2層の見送り記録

**層1: 非エントリー候補** - 分析中に ★ でウォッチしたがエントリーしなかった銘柄
- 入力タイミング: なし(★ でメモを書いた時点でその銘柄別メモが記録。エントリーアクションで別銘柄が確定した時点で自動的に層 1 扱いになる)
- 保存先: `Session.candidates[].memo`(銘柄別メモ、自由記述)
- 見送り理由の独立フィールドは設けない。書きたければ銘柄別メモに自由記述([§6.3.2](./06-session-screen.md#632-エントリーと非エントリー候補))

**層2: セッション見送り** - そのセッションではどの銘柄にも入らないと判断
- 入力タイミング: セッション画面の「見送り」アクション([§6.3.3](./06-session-screen.md#633-セッション見送り))
- 保存先: `Session.final_decision.skip_reason`(自由記述、任意) / `considered_styles`(検討したスタイル、任意)
- 任意入力(素早く見送りたいユーザを妨げない)

## 9.2 事後データ表示
セッション終了時、見送ったペアについて以下を表示:
- **3段階の事後チャート**(10本先 / 50本先 / 200本先)
- 各期間の最大上昇幅・最大下落幅
- 買い/売り想定時の推定成績
- 実際にエントリーしたペアとの比較

## 9.3 事後評価の表示方針

事後 OHLC から **pips 数値と事後チャートのみを表示** し、「機会損失 / 正解 / どちらでも」等のラベル判定は採用しない([principles/no-tags.md](./principles/no-tags.md))。理由:

- **結果バイアスを避ける**: [§11.9](./11-ai-analysis.md#119-訓練価値保全のためのプロンプト設計方針) は「結果論に引きずられるため AI には機械判定ラベルを送らない / 再発明もさせない」と定めている。同じ論理は訓練を受けるユーザー自身にも当てはまる。「正解」「機会損失」を画面に出せば、次セッションの判断軸が「ラベルで高得点を狙う」に歪む
- **閾値は訓練スタイルに依存**: 「`opportunity_pips` = 20pips」「`lookahead_bars` = 50 本」といった閾値の意味はスタイルによって変わる。機械的に「正解」を定義すると、スタイル選定の文脈を潰す
- **解釈が訓練の本体**: pips 数値と事後チャートを見て自分で評価すること自体が訓練。閾値ラベルは「書かずに済ませる逃げ道」として機能し、自由記述メモ方針と整合しない

**算出する数値**(表示のみ、集計しない — [principles/no-aggregation.md](./principles/no-aggregation.md))

- 起点(`presented_at` の close)から [§9.2](#92-事後データ表示) の 3 段階内の **最大上昇 pips** / **最大下落 pips**
- 想定方向は保存しないため、上昇・下落を両方そのまま表示する(買い想定/売り想定の推定成績はユーザーが自分で読む)

## 9.4 単一セッションの事後振り返り表示

§9.3 の事後 pips は **1 セッション単位の振り返り表示** にのみ使う。横断集計(見送り後の値動き統計・機会損失率 等の分母を取って平均する指標)も、単一セッションでの機械判定ラベルも提供しない([principles/no-aggregation.md](./principles/no-aggregation.md) / [principles/no-tags.md](./principles/no-tags.md))。

セッション終了後の振り返り画面では、そのセッションに限って以下を表示する:
- 見送った候補ごとの最大上昇 pips / 最大下落 pips(ラベルなし)
- 事後 OHLC を用いた 10/50/200 本先チャート([§9.2](#92-事後データ表示))
- エントリー済みの場合: `pips_pnl`、SL/TP 到達位置

数値はその場で確認して自分で解釈し、必要なら次のセッションの判断材料にする。複数セッションを横断した勝率・期待値・傾向分析は AI・ローカル共に持たない。
