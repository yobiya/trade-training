# §9. 判断結果の事後確認機能

← [仕様書インデックス](./README.md)

---

本章では、エントリー・見送り後にその判断結果を確認するための機能を定義する。
評価軸は **R:R 比率**(R = エントリー時の SL 幅)。pips は補助表示として併記する([設計方針](#93-事後評価の表示方針) 参照)。

## 9.1 判断の記録

セッション内で発生しうる判断は 3 種類:

**層 1: 非エントリー候補** - 分析中に ★ でウォッチしたがエントリーしなかった銘柄
- 入力タイミング: なし(★ でメモを書いた時点でその銘柄別メモが記録。エントリーアクションで別銘柄が確定した時点で自動的に層 1 扱いになる)
- 保存先: `Session.candidates[].memo`(銘柄別メモ、自由記述)
- 見送り理由の独立フィールドは設けない。書きたければ銘柄別メモに自由記述([§6.3.2](./06-session-screen.md#632-エントリーと非エントリー候補))

**層 2: セッション見送り** - そのセッションではどの銘柄にも入らないと判断
- 入力タイミング: セッション画面の「見送り」アクション([§6.3.3](./06-session-screen.md#633-セッション見送り))
- 保存先: `Session.final_decision.skip_reason`(自由記述、任意)
- 任意入力(素早く見送りたいユーザを妨げない)

**エントリー** - セッション画面でエントリーアクションを取り、`Trade` レコードが生成されたもの
- 層 1 / 層 2 と並んで本章の事後確認対象になる

## 9.2 事後観察データの種類

タイミングごとに **何を見せるか / 見せないか** を以下のように定める。「見せない」は [principles/no-future-info.md](./principles/no-future-info.md)(判断時点で実トレードでは知り得ない情報は訓練画面にも出さない)に従う。

| タイミング | 表示するもの | 表示しないもの |
|---|---|---|
| 見送り後(層 1 / 層 2) | 3 段階の事後 OHLC(10 / 50 / 200 本先)+ 最大上昇 pips / 最大下落 pips(SL 未確定のため R 計算なし) | ラベル判定 / 横断集計 |
| エントリー・保有中 | 現在価格 / 現在含み R / エントリー後の経路(チャート) | **MFE / MAE の自動要約** / 未来ローソク |
| エントリー・決済後 | MFE / MAE(保有期間内) / 続き観察 OHLC / 実損益 R | ラベル判定 / 結果論の評価文言 |

**保有中に MFE / MAE を出さない理由**: 実トレードの取引画面に MFE/MAE を自動表示する仕組みはない。訓練画面だけが出すと判断軸が「ツールが示した極値」に寄り、実戦復帰時に情報ギャップが生じる([principles/no-future-info.md](./principles/no-future-info.md))。

## 9.3 事後評価の表示方針

事後 OHLC から事後の値動きを表示する。**エントリー済みは R 単位 + pips 補助**、**見送り(層 1 / 層 2)は pips のみ**(SL が未確定で R 換算の信頼度が低いため、見送り時は代理 R を計算せず絶対値 pips のみとする)。

「機会損失 / 正解 / どちらでも」等のラベル判定は採用しない([principles/no-tags.md](./principles/no-tags.md))。理由は前節の結果バイアス回避と同じ。

### R の基準

- **エントリー時**: 実 SL 幅を 1R とする(= `|entry - sl|` の pips 換算値)
- **見送り・候補見送り時**: SL が未確定のため R 計算は行わず、絶対値 pips のみ表示する

### 算出する数値(表示のみ、集計しない — [principles/no-aggregation.md](./principles/no-aggregation.md))

起点(見送りなら `presented_at` の close、エントリーなら `entry_price`)から [§9.2](#92-事後観察データの種類) の対象期間内の以下を算出:

- **エントリー時**: 最大順行 R / 最大逆行 R(direction を考慮)+ 同 pips 補助
  - 最大順行 R = `max(window.high - ref) / R_unit`(買い時)または `max(ref - window.low) / R_unit`(売り時)
  - 最大逆行 R = 同上の逆方向
- **見送り時**(想定方向なし): 上昇側と下落側を別々に「最大上昇 pips / 最大下落 pips」として並べる(絶対値、R 換算なし)

### pips 補助表示

R の括弧書きで絶対値 pips を併記する(例: `+1.5R (+30pips)`)。
目的はスプレッド影響の把握:

- R が同じでも pips が小さいとスプレッド比率が大きくなる
- 異なる銘柄・スタイル間で「スプレッド的に無理があったか」を読み取れる

## 9.4 単一セッションの事後振り返り表示

§9.3 の事後 R は **1 セッション単位の振り返り表示** にのみ使う。横断集計(勝率・機会損失率 等の分母を取って平均する指標)も、単一セッションでの機械判定ラベルも提供しない([principles/no-aggregation.md](./principles/no-aggregation.md) / [principles/no-tags.md](./principles/no-tags.md))。

セッション終了後の振り返り画面では、そのセッションに限って以下を表示する:

- 見送った候補ごとの最大上昇 pips / 最大下落 pips(SL 未確定のため R 換算は行わない、ラベル判定もなし)
- 事後 OHLC を用いた 10/50/200 本先チャート([§9.2](#92-事後観察データの種類))
- エントリー済みの場合: 下記 §9.5 参照

数値はその場で確認して自分で解釈し、必要なら次のセッションの判断材料にする。複数セッションを横断した勝率・期待値・傾向分析は AI・ローカル共に持たない。

## 9.5 エントリー結果の事後確認

エントリーが決済まで完了したセッションの振り返り画面では、以下を表示する:

- **実損益 R** = `(exit_price - entry_price) / (entry_price - sl_price) × sign(direction)`
  - 例: 買いエントリー、SL -20pips、TP +40pips、+30pips で決済 → 実損益 +1.5R
  - pips 括弧書き併記(例: `+1.5R (+30pips)`)

- **MFE(最大順行 R)** = 保有期間(entry_time 〜 exit_time)の M5 OHLC を走査し、順行側の最大幅を R 単位に換算した値
  - 決済タイミングの妥当性を振り返る材料(「MFE は +2R だったが +1.5R で決済した」等)

- **MAE(最大逆行 R)** = 同じく保有期間内の逆行側の最大幅
  - SL 幅の妥当性を振り返る材料(「MAE が 0.9R なら SL にギリギリ耐えた」等)

- **続き観察** = 決済後の OHLC を **追加 N 本分チャート表示**(▶ 続き観察 ボタン / §6.1)
  - 決済判断の妥当性を振り返る材料(「決済後さらに +1R 伸びた」「逆方向に戻った」等)
  - 要約数値は出さず、チャートをそのまま見せる(自動要約の粒度抑制)
  - N の初期値は 50 本(M5)。スタイルによって適切な値は変わる

表示はあくまで 1 セッション単位の振り返り用。横断集計・ラベル判定は行わない。
