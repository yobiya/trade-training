# §6. セッション画面(統合フロー)

← [仕様書インデックス](./README.md)

---

## 6.1 画面構成

セッションのすべてのフェーズ(分析 / 保有 / 振り返り)を 1 つの画面で処理する。画面遷移は伴わない。フェーズはセッションの状態(アクティブ Trade の有無 / skip 済か)から自動的に導出する。

- **ヘッダー**: 戻る、**セッション名(任意・自由記述、いつでも編集可)**、銘柄セレクタ(分析中のみ操作可、保有中以降は現在銘柄を固定表示)、TimeframeSelector、📝 メモボタン
  - セッション名は手法やテーマを識別するための自由記述。毎回違うニュアンス(「ダブルトップ逆張り」「指標前のトレンド継続」等)を残せる。一覧画面で銘柄・日付・損益 R と並べて表示される
  - **銘柄セレクタ(分析中のみ)**: 28 ペアの一覧操作を高速化するため、以下の要素を 1 行に並べる
    - **ALL / ★ モード切替ピル**: ALL は §2.8 全ペアを表示、★ は本セッションで ★ を付けた候補銘柄のみに絞り込む。★ 数が 0 の時は ★ ボタンは disabled、選択中に 0 件になった場合は ALL に自動フォールバック
    - **prev / next ボタン (`‹` / `›`)**: 現在モードのリスト内で前後の銘柄に循環移動
    - **絞り込み済みドロップダウン**: ★ 付き銘柄は先頭に `★` を付与
    - **★ トグルボタン**: 現在表示中の銘柄に対する ★ の付与 / 解除(分析中フェーズでのみ表示)
    - **キーボードショートカット**(分析中のみ、入力欄フォーカス中は無効): `[` = 前の銘柄、`]` = 次の銘柄、`F` = ALL/★ モードトグル、`S` = 現在銘柄の ★ トグル、`M` = メモパネル
- **メインエリア**: マルチタイムフレーム縦積みチャート([§5.1](./05-chart.md#51-表示))+ 描画オーバーレイ
- **右サイドバー**: フェーズに応じて中身が変わる
- **メモはパネル切替式**([§7.3](./07-memo.md#73-ui-メモパネル切替式)): `M` キー / 📝 ボタンで全画面オーバーレイ、`Esc` で閉じる。全フェーズで利用可

| フェーズ | 判定 | サイドバー主要要素 |
|---|---|---|
| 分析中 | アクティブ Trade なし、skip 未確定 | IndicatorPanel / DrawingTools / TradePanel(現在値表示 + チャート上 SL/TP 配置 + 単一エントリーボタン、§7.4) / ▶足送り / 見送り |
| 保有中 | アクティブ Trade あり | 保有中の情報 / IndicatorPanel / DrawingTools / TradePanel(exit 入力) / ▶足送り |
| 振り返り | 決済済 Trade あり or skip 済 | Trade 結果(実損益 R / pips 補助) / PostReviewPanel(MFE・MAE・事後 R / [§9.5](./09-post-review.md#95-エントリー結果の事後確認)) / IndicatorPanel / DrawingTools / ▶続き観察 |

ヘッダーに **状態バッジ(進行中 / 決着済み)** を表示する([§4.2.1](./04-session-flow.md#421-セッション状態モデル進行中--決着済み))。決着済みになってもメモ・描画の追記・編集は引き続き可能で、ボタン類のロックはしない。

セッションの破棄ボタンは画面に置かない。削除は OS / Dropbox 上のディレクトリ操作で行う([§13](./13-data-storage.md))。

**「銘柄選定」というステップを独立した状態として持たない**。ユーザーが分析しながら気になる銘柄を切り替え、条件が揃った時点でエントリーアクションを取ることで、そのセッションの対象銘柄が確定する。銘柄を先に決めて後から分析するのではなく、**分析の結果としてエントリー銘柄が決まる**(= 実トレード思考と一致)。

描画・インジケーターは全フェーズで使える。描画は銘柄別([§5.3](./05-chart.md#53-描画ツール))、インジはセッション内でグローバル(フェーズ遷移しても継続)。

## 6.2 銘柄セレクタ(ヘッダ統合)
- 28 ペアもの銘柄をサイドバーの縦リストに並べると一覧性 / 操作性が悪いため、ヘッダにコンパクトに集約する形を採る
- 構成要素は §6.1 ヘッダー記述参照(ALL/★ ピル + prev/next + ドロップダウン + ★ トグル + キー shortcut)
- **直近変動率・通貨強弱などの要約指標は表示しない**。ユーザーが詳細チャートを見て自分の目で判断する(§6.4)
- 保有中・振り返りフェーズでは銘柄は確定済のため、ヘッダは現在銘柄のテキスト表示に切り替わる(切替操作不可)

## 6.3 候補 ★ の役割

**★ は「銘柄別メモを残す入れ物」としての機能のみ**。選定確定という独立したステップは存在しない。エントリーしたい銘柄が定まれば、分析中にその場でエントリーアクションを取ることで銘柄が確定する。

### 6.3.1 候補の追加とメモ
- 「★ 候補に追加 / 解除」で現在表示中の銘柄を `SessionCandidate` に追加。銘柄別メモの入れ物ができる
- 銘柄別メモ(自由記述)は新規候補作成時に見出しテンプレート([§7.2.3](./07-memo.md#723-見出しテンプレート初期値))が初期挿入される
- 複数銘柄を比較している途中の気づきをその場で書き残すために使う

### 6.3.2 エントリーと非エントリー候補
- エントリーアクションを取ると、対象銘柄の `SessionCandidate` が自動生成(未作成なら)され、`is_selected = True` が立つ
- 同セッション内で他に ★ 済みの銘柄がある場合、それらは **非エントリー候補**(層 1 として振り返り対象)となる
- 非エントリー候補に対する「見送り理由」を確定前に入力させる確認フォームは **設けない**。書きたければ各銘柄の銘柄別メモに自由記述する([§7.2.1](./07-memo.md#721-銘柄別メモ))
- これにより「エントリーする」アクション = セッションの対象銘柄確定、という 1 ステップで済む

### 6.3.3 セッション見送り
- 「見送り」ボタンで `SessionFinalDecision.has_entry = False` を記録し、セッションを見送り状態にする(= 層 2)
- 理由は任意記述
- 見送り後も振り返りフェーズに移行し、PostReviewPanel で事後値動きを確認できる

## 6.4 通貨強弱・変動率の自動表示について

**自動算出・ダッシュボード・構造化入力・銘柄別の直近変動率表示は実装しない**。ユーザーが生チャートから自分の目で強弱を判断する訓練方針を貫く。

- 直近変動率(% 表示)は「最近動いた」という事実しか伝えず、これから優位性があるかの情報にならない。目に入るとその銘柄へのバイアスが生じるため、サイドバーには表示しない

- 読みは横断メモ([§7.2.2](./07-memo.md#722-横断メモ))に**自由記述**で記録する
- 構造化入力(通貨別数値・選択式タグ等)は不採用([principles/no-tags.md](./principles/no-tags.md) 参照)
- 横断メモは 1 セッション単位の AI 観察([§11](./11-ai-analysis.md))の視点セクションの対象になる。横断集計は採用しない([principles/no-aggregation.md](./principles/no-aggregation.md))
