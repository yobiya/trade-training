# §16. 段階リリース計画

← [仕様書インデックス](./README.md)

---

## Phase 1: MVP基盤構築 (B の最小版)
- **モノレポ・uvワークスペース初期化**: ディレクトリ構成のセットアップ、`pyproject.toml` / `uv.lock` 初期化、npm workspace設定
- **shared-schema 初期版**: 最低限のテーブル定義(キャッシュテーブル含む)、alembic初期マイグレーション
- **market-data 第一版**: MT5Provider、オンデマンド取得、SQLiteキャッシュ層。MT5で主要銘柄のM5 5年分が取得可能かの検証タスクを含む(取得できない場合はDukascopy等への切り替え検討)
- **アプリB (trade-trainer) MVP**:
  - 単一時間軸チャート表示・足送り機能(market-data経由でデータ取得)
  - 買い/売り/見送り + SL/TP設定
  - 裁量決済

## Phase 2a: 分析体験の本格化
**出口条件**: 実戦レベルの分析環境でトレードできる
- `common-ui-lib` 初版(チャート、メモフォーム、描画ツール雛形)
- マルチタイムフレーム表示
- 描画ツール(水平線・トレンドライン・フィボナッチ)
- インジケーター(SMA/EMA/RSI)
- シナリオメモ機能(**最小限**: メモ本文 + エントリー根拠)

## Phase 2b: 訓練コンセプトの実装
**出口条件**: 訓練アプリ固有の価値(時間起点・スタイル・3シナリオ)が揃う
- 時間起点の銘柄選定フロー(左サイドバー銘柄一覧、詳細チャート切替、★ + 候補メモのウォッチリスト)
- **トレードスタイル管理**: スタイル定義画面、プリセット、エントリー時のスタイル選択UI、見送り時の検討スタイル記録
- シナリオメモ**拡張**(環境認識、相場観、指標認識、代替シナリオ、スタイル選定理由、銘柄選定理由、比較候補の総評)
- 波動ラベリング支援

## Phase 2c: 振り返り・事後評価・経済指標
**出口条件**: 1 セッション単位で判断の質を振り返れる / 経済指標を意識した訓練ができる
- 判断結果の事後確認機能(§9): 見送り後 3 段階の事後チャート + 最大順行/逆行 R(pips 補助) / エントリー後 MFE・MAE・続き観察 OHLC・実損益 R — **1 セッション内表示のみ、ラベル判定なし、横断集計なし、保有中の自動要約なし([principles/no-future-info.md](./principles/no-future-info.md))**
- **経済指標表示(星3・縦線+シェーディング)** + market-data に経済指標取得機能追加、CLI日次バッチ設定

## Phase 3: 運用性(B の完成)
- 設定画面からの market-data 経済指標手動更新ボタン
- **セッション情報のファイル化**(ver 1.45、[§13](./13-data-storage.md) / [§17](./17-data-model.md)):
  - 旧 6 テーブル(sessions / session_candidates / session_final_decisions / trades / holding_memos / drawings)を `data/sessions/{dir}/` ファイル群に移行
  - トレードスタイルも `data/trading-styles/{id}.md` に移行
  - 状態モデルを「進行中 / 決着済み」の 2 状態に整理(`is_suspended` 撤廃)
  - AI 分析結果ディレクトリをセッション配下(`data/sessions/{dir}/ai_analysis/`)に統合
  - Dropbox 同期前提の運用に対応(atomic write、conflict 除外、id 重複検出)
- スマホUI最適化

※ 横断集計(勝率・期待値・スタイル別成績等)は採用しない([principles/no-aggregation.md](./principles/no-aggregation.md))。一方、**個別セッションのファイル単位での永続保持と振り返り**は ver 1.45 で許容方針に変更。

## Phase 4: AI 分析機能 (§11)
- **§11 AI 分析** のファイル保存(`data/sessions/{dir}/ai_analysis/` 配下、[§11.7](./11-ai-analysis.md#117-分析結果の永続化ファイルストレージ))+ 任意の送信前プレビュー
- チャート画像レンダリング、メタデータ・メモ送付、Markdown レポート表示
- **1 セッション単位のみ**。AI 分析結果はセッションディレクトリ配下に保存され、ユーザーが OS / Dropbox 上でセッションディレクトリを削除すれば同時に消える([§13](./13-data-storage.md))
- コスト管理(月額バジェット・利用量ダッシュボード)は採用しない([§11.6](./11-ai-analysis.md#116-データ送信の調整キャッシュ時刻整形)、ver 1.47)
- 専用の集計アプリ(trade-analyzer)は持たない。AI 分析は trade-trainer 本体の機能として統合する
- **データ蓄積を前提としないため、Phase 1-3 運用後の待ち時間は不要**。実装が揃えばすぐに利用可能

## Phase 5: リアルトレードアプリ・デモ版 (C の実装、デモ口座)
- **アプリC (trade-live) 第一版**:
  - common-ui-lib を利用したUI構築(配色は警告色系)
  - MT5リアルタイムデータ表示
  - MT5**デモ口座**への発注機能
  - ロット自動計算・リスク管理
  - 保有中の気づきメモ(任意、タイムスタンプ付き自由記述)
  - 発注安全機構(二段階確認のみ、発注前入力は全て任意)
  - ポジション継続管理(オープンポジション一覧、当初メモ参照、気づきメモ記録)
  - オンデマンド AI 分析([§11](./11-ai-analysis.md)、ユーザー明示トリガーのみ。trade-live 本体から呼び出し、別アプリは介さない)

## Phase 6: リアルトレードアプリ・実口座 (C の本運用)
- 実口座接続(最小ロットからスタート)
- 通常ロットでの本格運用
- trade-live → trade-trainer の復習ボタン連携(リアルで遭遇した局面をその場で trainer の新規セッションに引き継ぐ。過去の蓄積ではなく、都度の橋渡しとして機能)
- リアル特有の 1 セッション振り返り表示(保有時間・当初メモと決済結果の対比。横断集計・履歴参照は持たない)
