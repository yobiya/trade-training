# §7. メモ機能(自由記述ベース、2 本立て)

← [仕様書インデックス](./README.md)

---

以下はすべて**トレーニング(trainer)モード専用**。リアル(journal)モードでは発注前入力に必須項目を設けない([§12.5](./12-live.md#125-発注前入力の任意性と安全機構) 参照)。

## 7.1 設計方針: フォーマットを押しつけない

**集計を行わない方針([principles/no-aggregation.md](./principles/no-aggregation.md))** と整合して、メモ入力もフィールド分けやフォーマットを強制しない。

- 「環境認識」「相場観」「指標認識」「銘柄選定理由」「メインシナリオ」「代替シナリオ」「エントリー根拠」などの項目を **別々の必須フィールドに分けない**
- 訓練の目的は「思考プロセスを言語化すること」。項目分けは書く内容のヒントになり得る一方、必須化すると「埋めるための作文」になって言語化の質が落ちる
- ただし実運用では **銘柄固有の観察と銘柄横断の観察が明確に別のもの** として生じる(例: 「USDJPY の H4 チャート形状」と「ドル全面高で JPY クロスが優位」)。この 2 つを 1 本のテキストに混在させると読み返しにくいため、メモを **2 本立て** にする

**Markdown 見出しテンプレートの扱い**: 書く内容のヒントとして、空のメモに見出しテンプレート(例: `## 相場観` `## シナリオ` 等)をあらかじめ初期挿入することは許容する([§7.2.3](#723-見出しテンプレート初期値))。これは「フィールド分け・必須化」とは異なり、**見出し自体はユーザーが自由に削除・編集・並び替えできる単なるテキスト初期値** であるため、本節の「フォーマット強制しない」方針と矛盾しない。見出しが不要なら全削除して空白からスタートしてよい。

## 7.2 メモの種類(2 本立て)

| 種類 | 紐付き | 用途 | データモデル |
|---|---|---|---|
| **銘柄別メモ**(§7.2.1) | 各候補銘柄 | その銘柄のチャート観察・トレンド・根拠・個別判断 | `SessionCandidate.memo`(既存) |
| **横断メモ**(§7.2.2) | セッション全体 | 通貨強弱・銘柄比較・相場観・シナリオ・見送り理由・決済所感・振り返り | `Session.note` TEXT(既存) |

判断に迷う書き分けはしない:
- 「この銘柄」の観察は銘柄別メモ
- 「銘柄横断・セッション全体」の観察は横断メモ
- どちらか曖昧なら横断メモに書いて問題ない

### 7.2.1 銘柄別メモ

- セッション画面の ★ 候補追加時から書き始められる([§6.3.1](./06-session-screen.md#631-候補の追加とメモ))
- 候補化された時点で見出しテンプレート([§7.2.3](#723-見出しテンプレート初期値))が初期挿入される
- エントリー後の保有・振り返りフェーズでも編集継続可能(銘柄は同じ画面内でドロップダウンから選べる)
- エントリーした銘柄以外の非エントリー候補のメモは 層 1 振り返り対象([§6.3.2](./06-session-screen.md#632-エントリーと非エントリー候補))としてそのまま保持される
- 保有中の気づきメモ([§12.6](./12-live.md#126-心理状態の記録))は journal(リアル)固有なので別管理

### 7.2.2 横断メモ

- セッション 1 つにつき 1 つの自由記述テキスト
- セッション開始からセッションがファイルシステムから削除されるまで一貫して編集可能(決着済みになっても編集可、[§4.2.1](./04-session-flow.md#421-セッション状態モデル進行中--決着済み))
- 空の状態で新規セッションを開始すると、見出しテンプレート([§7.2.3](#723-見出しテンプレート初期値))が初期挿入される。不要なら削除してよい
- 1 セッション単位の AI 分析([§11](./11-ai-analysis.md))に全文送付される
- **決着遷移トリガー**: トレード決済済み or 見送り確定済みの状態で、本メモが空文字以外に更新されると、セッションが「決着済み」に自動遷移する([§4.2.2](./04-session-flow.md#422-決着遷移条件))。中身が中途半端でも 1 文字以上あれば該当する

### 7.2.3 見出しテンプレート(初期値)

空のメモを開いた時の初期テキストとして、Markdown 見出しテンプレートをあらかじめ挿入する。書く内容のヒントとして機能する一方、**必須フィールドではない**: ユーザーは自由に削除・追加・並び替えできる。

**銘柄別メモの既定テンプレート:**

```
## チャート観察(TF・トレンド・形状)

## サポレジ・水準

## 根拠 / 狙いどころ

## SL / TP の考え
```

**横断メモの既定テンプレート:**

```
## 相場観・通貨強弱

## 銘柄比較

## シナリオ(メイン / 代替)

## 判断

## 決済・振り返り
```

**保存場所:**

テンプレート本体は **リポジトリ内の Markdown ファイル** で管理する。テキストエディタ + git で編集・履歴管理できるよう DB ではなくファイルを採用する(個人運用前提)。

```
data/memo-templates/
├── candidate.md       # 銘柄別メモ初期テンプレ
└── session-note.md    # 横断メモ初期テンプレ
```

- 編集はテキストエディタで直接行い、変更は git でそのままコミットする(個人運用前提)
- バックエンドは起動時に 1 回読み込んでメモリ保持。編集の反映にはバックエンド再起動が必要
- 設定画面 UI は提供しない(ファイル直接編集に置き換え)
- 環境変数 `MEMO_TEMPLATES_DIR` で別ディレクトリを指定可能(開発環境向け)

**運用ルール:**

- テンプレートは **新規セッション開始時 / 新規候補追加時** にのみ挿入される。既に記入があるメモには再挿入しない(既存内容を消さない)
- 該当 Markdown ファイルが存在しない・空ファイルの場合は **テンプレ無効** として扱う(挿入しない)
- 見出し未使用のセクションは削除推奨(空の見出しが残っていると AI 分析時のノイズになる)

## 7.3 UI: メモパネル切替式

チャートとメモを同時表示するとチャートが狭くなるため、**メモパネルはサイドバーに常駐させず、ワンアクションで開閉する** 方式を採用する。

### 7.3.1 切替トリガー

| 方法 | 動作 |
|---|---|
| ヘッダー右端の「📝 メモ」ボタン | クリックでメモパネルをトグル表示 |
| キーボードショートカット `M` | メモパネル開閉トグル |
| `Esc` キー | メモパネル表示中に押すと閉じる |

### 7.3.2 メモパネルの構造

チャート領域全体を覆うパネル(モーダル風オーバーレイ)。縦に 2 段:

```
┌─────────────────────────────────────────────┐
│ 📝 メモ             [× 閉じる / Esc]        │
├─────────────────────────────────────────────┤
│ 銘柄別メモ: [USDJPY ▼]                      │
│ ┌─────────────────────────────────────────┐ │
│ │ ドル強度明確、H1 でダブルトップ形成中   │ │
│ └─────────────────────────────────────────┘ │
├─────────────────────────────────────────────┤
│ 横断メモ(相場観・比較)                    │
│ ┌─────────────────────────────────────────┐ │
│ │ ドル全面高、クロス円は円売り混じりで    │ │
│ │ USDJPY が優位。EURJPY は方向感弱い      │ │
│ └─────────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
```

- 上段: **銘柄別メモ**。ドロップダウンで対象銘柄を切替可
- 下段: **横断メモ**(セッション全体で 1 つ)
- 両方とも自由記述 textarea。編集は即時保存(フォーカス離脱 or 一定間隔)

### 7.3.3 画面別の銘柄別メモ対象

| フェーズ | 上段(銘柄別メモ)の対象 |
|---|---|
| 分析中 | 現在チャート表示中の銘柄が初期選択。候補以外でも表示中の銘柄のメモに切り替わる |
| 保有中 / 振り返り | エントリー銘柄を既定表示。ドロップダウンから層 1(非エントリー候補)のメモも閲覧・編集可 |

## 7.4 エントリー時の必須項目(最小限)

「ボタンを押したら即発注できる」状態にするための数値情報のみ必須。思考・根拠はメモに書く(銘柄別または横断)。

| 項目 | 必須 | 入力方法 | 備考 |
|---|---|---|---|
| エントリー価格 | (自動) | 現在値で固定 | 成り行きエントリー想定。手入力不可 |
| 方向 | (自動) | SL の位置から導出 | `SL < 現在値` → BUY、`SL > 現在値` → SELL |
| SL | ✓ | チャート上をクリックして配置 | 配置後に解除・再配置可能 |
| TP |  | チャート上をクリックして配置(任意) | SL と反対側であること(同側に置こうとするとエラー) |

ユーザー操作は「SL を配置 → (任意で)TP を配置 → 単一の『エントリー』ボタンで確定」の 3 ステップ。BUY / SELL を選ぶ二択ボタンは持たない(SL の位置が方向そのものを表す)。

※ エントリー根拠・シナリオ等はメモに自由記述。フォームに独立フィールドは設けない。

## 7.5 見送り時の必須項目

なし。理由を残したい場合は見送り確定時の自由記述、もしくは横断メモで自由記述する。

## 7.6 判断フロー(UI 流れ)

1. **分析段階**: メモパネル(M キー or ボタン)で銘柄別メモ・横断メモを書きながら、閉じてチャートを精査する往復
2. **判断アクション**: 下部バーの「エントリーする」または「見送る」を選択
   - エントリー: チャートに SL(必須)/ TP(任意)を配置 → 方向自動判定 → 「エントリー」ボタンで確定([§7.4](#74-エントリー時の必須項目最小限))
   - 見送り: 確定で見送り記録
3. **保有中 / 決済後**: メモパネルを開いて気づき・決済所感を追記
4. **振り返り**: 横断メモに時間を置いて追記可能。決済直後は興奮状態で精度が低いため、冷静になってから書くのを推奨

## 7.7 決済・振り返りの扱い

- **決済理由**(TP 到達 / SL 到達 / 裁量判断)は Trade レコードに数値として残す([§9.5](./09-post-review.md#95-エントリー結果の事後確認) の事後確認で参照)。言葉での補足は横断メモに書く
- **振り返り**(実展開とシナリオの対応、改善点、体調・心理・判断の質への気づき)は横断メモに追記。独立フィールドは設けない
- **後日追記**: セッションは決着済みになっても永続化され、メモ・描画は引き続き編集可能([§4.2.1](./04-session-flow.md#421-セッション状態モデル進行中--決着済み) / [principles/no-aggregation.md#103-セッションのライフサイクル](./principles/no-aggregation.md#103-セッションのライフサイクル))

## 7.8 タグ・構造化選択式入力を採用しない方針

→ [principles/no-tags.md](./principles/no-tags.md) に集約。本方針は §6.4 / §7.1 / §9.1 / §11 等から参照される横断方針のため、ハブファイルに切り出している。
