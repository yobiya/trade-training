## 5.1 表示

### 5.1.1 マルチタイムフレーム縦積み
- 全 TF (M5/M15/H1/H4/D1/W1/MN1) のチャートを**画面幅フルで縦に並べる**
- 各 TF を個別に**表示/非表示切替**できる(不要な TF を隠して集中できる)
- 並び順: **TF 固定順 (M5 → M15 → H1 → H4 → D1 → W1 → MN1)** で常に上から並ぶ。フォーカス切替で順序が変わらないため空間記憶が安定する
- 表示 TF 数に応じて各チャートが高さを等分する(減らすほど各チャートが大きくなる)
- セッションの `current_position` は全 TF 共通、足送りは全 TF 同期
- 未来部分は隠蔽、ローソク足を1本ずつ進行可能
- 「+1本」「+5本」の **「1 本」はフォーカス TF の 1 バー** を指す(フォーカス TF が M15 なら +1 本 = M5 換算 +3 本 = +15 分。M5 がフォーカス TF のときだけ +1 本 = +5 分)。これによりユーザーが見ているメイン足が必ず 1 本進む。フォーカス TF は §5.1.5 参照
- **各 TF の最新バー(進行中)は「一つ下の TF」を集約して算出**する(M15 の最新バーは M5 から、H1 の最新バーは集約済み M15 から、… MN1 の最新バーは集約済み W1 から)。これにより `current_position` 以降の未来情報が混入しない

### 5.1.2 時間軸・クロスヘア同期
- **クロスヘア同期**: ある TF でマウスを動かすと、他の TF 上にも同じ時刻の縦線が表示される
- **時間軸パン同期**: ある TF をスクロール・ズームすると、他の TF も対応する時間帯に追従
- 効果: 複数 TF 間の対比(D1 の戻り = H1 の押し目等)を視線移動だけで行える

### 5.1.3 ホイール・ドラッグ操作
縦スクロールとチャートのズームが衝突するため、以下の操作体系とする。

| 操作 | 振る舞い |
| --- | --- |
| ホイール (素) | ページの**縦スクロール**(他 TF を見る) |
| Ctrl + ホイール | そのチャートの**時間軸ズーム** |
| 時間軸のドラッグ | 時間軸のスケール変更 |
| 価格軸のドラッグ | 価格軸のスケール変更 |
| チャート中央のドラッグ | 時間軸のパン |
| タッチ: ピンチ | ズーム |
| タッチ: 1 本指スワイプ | ページスクロール |

- 素のホイール = ページスクロールは Web の慣習(Ctrl + ホイール = ズームは Figma/Google Maps 等でも共通)に一致させ、学習コストを下げる
- 実装では lightweight-charts の `handleScale.mouseWheel` を無効化し、上位 DOM で `ctrlKey` 判定付きの wheel ハンドラを自前実装する

#### 表示本数の維持
- 銘柄切替時に表示本数(可視範囲のバー数)が毎回変わらないよう、**TF ごとに「直近の表示幅(visible logical range の `to - from`)」をセッション内メモリで保持**する
- 銘柄切替・初回表示・初期マウントは「右端を最新バーに合わせ、メモリ上の幅で `setVisibleLogicalRange`」する。`fitContent()` の自動全件表示はしない
- ユーザーが Ctrl+ホイール / 時間軸ドラッグで幅を変えた値はそのまま次の銘柄に引き継がれる(同 TF 内)
- 既定値は **全 TF 共通の `DEFAULT_VISIBLE_BARS = 150`**。縦積みマルチ TF でローソク幅 / 表示密度が揃うようにするため、TF 毎に変えない。メモリ未確定時のフォールバックとして使用

### 5.1.4 その他
- **時刻表示は JST**: 時間軸ラベル、クロスヘア、ツールチップ全て JST 表示(内部データは UTC 保持、表示時のみ変換)
- 表示 TF の選択状態はセッションまたはユーザ設定として保存する(離脱・再入場で復元)

### 5.1.5 フォーカス TF
- 縦積みチャートの中で **「いま操作中心となる TF」= フォーカス TF** を 1 つ持つ。advance(+N 本)・SL/TP 配置・描画作成・エントリー時の `entry_tf` 記録は全てフォーカス TF を基準にする
- **クリックで明示選択**: TF チャート上を 1 度クリック / マウスダウンするとその TF がフォーカスになる。次に別の TF をクリックするまで保持され、マウスホバーでは変わらない(操作中心がぶれないため)
- **ハイライトのみ**: フォーカス TF はチャート枠線・背景の強調表示で示す。スタックの並び順(§5.1.1)は変わらない
- **初期値**: セッション初回マウント時は M5。銘柄切替・TF 表示 ON/OFF・フェーズ遷移ではリセットしない(直前のフォーカスを保持)
- **フェーズ別挙動**:
  - 分析中: 自由切替。クリックで focus、各種操作はフォーカス TF を中心に動く
  - 保有中・振り返り: フォーカスは自由に切替可能(チャートを見比べる / 描画したい TF を選ぶ)。advance も「フォーカス TF の N バー」単位で進む
  - **マーカー(エントリー / 決済の三角)** の表示対象 TF は `Trade.entry_tf`(エントリー時に固定された TF)であり、後からフォーカスを変えても移動しない([§5.5.4](#554-縦マーカーentry_tf-のチャートのみ))

## 5.3 描画ツール
| ツール | 実装 | デフォルト表示範囲 |
|---|---|---|
| 水平線 | ライブラリ標準 | 全時間足 |
| トレンドライン | ライブラリ標準 | 全時間足 |
| フィボナッチリトレースメント | プラグイン自前実装 | 作成時間足のみ |

- **銘柄別の紐付け**: 描画は作成時の銘柄(`symbol`)に紐付き、`SessionPage` で銘柄を切り替えるとその銘柄の描画だけが表示される。分析中に USDJPY に引いたラインが EURJPY に重ねて表示されることはない。エントリー後は該当銘柄の描画だけが自動的に読み込まれる。
- **作成時間足の自動記録**: 描画対象となったチャートの時間足を `timeframe` として保存する。マルチタイムフレーム分析において「どの時間足を見て引いたライン/カウントか」を後から識別できる。
- **表示対象時間足の個別制御**: `visible_on_timeframes` で表示する時間足を個別指定可能。上表は初期値であり、ユーザーが描画プロパティから変更できる(例: 特定のトレンドラインを上位足でも表示したい等)。
- **時間足の視覚的識別**: 描画の色を作成時の `timeframe` に応じて固定し、一目でどの時間足を見て引いた描画かを識別できるようにする。

  | TF  | 色                         |
  | --- | -------------------------- |
  | M5  | スカイブルー (#58a6ff)      |
  | M15 | ティール (#56d4dd)          |
  | H1  | 緑 (#26a69a)                |
  | H4  | 黄 (#e3b341)                |
  | D1  | オレンジ (#f0883e)          |
  | W1  | 紫 (#d2a8ff)                |
  | MN1 | マゼンタ (#ff79c6)          |

  - ツール種別は形状で判別する(水平線・線分・フィボナッチ)、色は時間足識別に専念する
  - 描画のホバー・選択時には端点付近に TF バッジ(例: `[H1]`)を表示し、色覚配慮の補助情報とする
  - ユーザーが `visible_on_timeframes` を変更して他の時間足に表示している描画も、色・バッジ共に **作成時の時間足** を示す(どの時間足の視点で引かれたかを保持する意図)
- **波動カウント支援**: エリオット波動のカウント UI は描画ツールではなく、§16 Phase 2b "波動ラベリング支援" の専用機能として別途提供する。
- **バー範囲外クリック**: 最終バーより右(whitespace)・最初のバーより左の位置でも描画が可能。lightweight-charts の `coordinateToTime` が範囲外で null を返すため、frontend 側で `coordinateToLogical` の値と TF 間隔(`TIMEFRAME_MINUTES`)から線形外挿した時刻を生成する。これによりトレンドラインやフィボナッチの頂点を未来側(右の余白)に置く運用が可能になる。MN1 は月毎日数差を 30 日近似で扱う。

## 5.4 経済指標表示
- **表示対象**: 重要度 **星3のみ**(設定で星2以上への拡張も可能な設計)
- **通貨フィルタ**: 表示中チャートの通貨ペアに関連する通貨 + USD(基軸通貨)
- **表示スタイル**:
  - **縦線マーカー**: 発表時刻に縦線(通貨別に色分け、ホバーで詳細表示)
  - **シェーディング**: 発表前後の時間帯(例: 前5分〜後30分)を背景色で塗り、ボラ上昇期間を可視化
- **時間軸上の情報公開ポリシー**(実戦で得られる情報と同じ扱い):
  - **未来の指標も表示**: 発表予定時刻、指標名、通貨、重要度、**予想値**、前回値はセッション現在時刻より未来でも表示(実戦で事前に知れる情報)
  - **実測値のみ発表時刻到達後に表示**: サプライズ度も実測値公開後に算出・表示
  - 縦線・シェーディングは未来分も含めて常時表示(次の指標までの残り時間を意識できる)
- **ホバー情報**: 指標名、通貨、発表時刻、実測値/予想値/前回値、サプライズ度(実測 vs 予想の乖離)

## 5.5 エントリー / 決済情報の表示

エントリー後・決済後の Trade 情報をチャート上に視覚化する。判断時点(分析中)に置いた SL/TP draft の表示([§7.4](./07-memo.md#74-エントリー時の必須項目))も同じ仕組みを使う。

### 5.5.1 表示要素

| 要素 | 種別 | 表示対象 TF | 色 |
|---|---|---|---|
| エントリー価格(`entry_price`) | 横線(priceLine、`Entry @ ...`) | 全 TF | 黄 (#e3b341) |
| SL 価格(`sl`) | 横線(`SL @ ...`) | 全 TF | 赤 (#ff5555) |
| TP 価格(`tp`) | 横線(`TP @ ...`) | 全 TF | 青 (#58a6ff) |
| 決済価格(`exit_price`) | 横線(`Exit @ ...`) | 全 TF | 緑(損益+)/赤(損益-) |
| エントリー時刻マーカー | 三角(BUY=上向き / SELL=下向き) | **エントリー TF のみ** | エントリー方向で色分け |
| 決済時刻マーカー | 三角(エントリーと反対向き) | **エントリー TF のみ** | 損益で色分け |

### 5.5.2 phase 別の表示マトリクス

| phase | entry 横線 | SL 横線 | TP 横線 | exit 横線 | entry マーカー | exit マーカー |
|---|---|---|---|---|---|---|
| **分析中(エントリー組み立て中)** | — | ✓(draft) | ✓(draft) | — | — | — |
| **保有中** | ✓ | ✓ | ✓ | — | ✓ | — |
| **振り返り**(エントリー済) | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| **振り返り**(見送り) | — | — | — | — | — | — |

### 5.5.3 横線(全 TF)
- エントリー後・決済後は `Trade.entry_price / sl / tp / exit_price` の値を全 TF の priceLine に注入する
- 描画オブジェクト由来の水平線(§5.3)と同じ priceLine 機構を使う
- ラベルに価格を併記(例: `Entry @ 150.123`、`SL @ 149.800`)
- 銘柄が確定済の保有中・振り返りでは銘柄切替不可([§6.1](./06-session-screen.md#61-画面構成))のため、表示銘柄と Trade 銘柄は常に一致する

### 5.5.4 縦マーカー(`entry_tf` のチャートのみ)
- エントリー / 決済時刻に三角マーカーを配置(lightweight-charts 標準 `setMarkers` API)
- マーカーが意味を持つのは **`Trade.entry_tf`(エントリー時に固定されたフォーカス TF)** のチャートだけ。他 TF はバー時刻と Trade 時刻が一致せずマーカーがズレるため非表示
- 後からフォーカス TF を変えてもマーカーは移動しない(エントリー時の TF 視点を保持する設計)
- マーカー位置は `entry_time` / `exit_time` を **`entry_tf` の bar 開始時刻に丸めた値**を使う
- shape: BUY エントリー = `arrowUp` (belowBar)、SELL エントリー = `arrowDown` (aboveBar)、決済はその逆向き

## 5.6 描画の保存
セッションごとに描画オブジェクトを保存し、後から視覚的に振り返り可能。

描画には以下のメタデータを保持する:
- `symbol`: 描画を行った銘柄(必須)。銘柄切替時に該当銘柄の描画のみ表示するために使う。
- `timeframe`: 描画を行ったチャートの時間足(必須)。判断根拠の時間軸コンテキストとしてAI分析にも活用する。
- `visible_on_timeframes`: 表示する時間足リスト(JSON 配列)。種類ごとの既定値は §5.3 参照、ユーザーが個別に変更可能。
