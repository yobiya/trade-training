## 5.1 表示

### 5.1.1 マルチタイムフレーム縦積み
- 全 TF (M5/M15/H1/H4/D1/W1/MN1) のチャートを**画面幅フルで縦に並べる**
- 各 TF を個別に**表示/非表示切替**できる(不要な TF を隠して集中できる)
- 並び順: **TF 固定順 (M5 → M15 → H1 → H4 → D1 → W1 → MN1)** で常に上から並ぶ。フォーカス切替で順序が変わらないため空間記憶が安定する
- 表示 TF 数に応じて各チャートが高さを等分する(減らすほど各チャートが大きくなる)
- セッションの `current_position` は全 TF 共通、足送りは全 TF 同期
- 未来部分は隠蔽、ローソク足を1本ずつ進行可能
- 「+1本」「+5本」の **「1 本」はフォーカス TF の 1 バー** を指し、**`current_position` をフォーカス TF の N 本目の境界へ進める**(境界アライメント。単純な時刻加算でも M5 換算でもない)。例: H1 focus で `current_pos = 10:15`(H1 バー [10:00, 11:00) の途中)から +1 本を押すと、新 `current_pos = 11:00`(次の H1 境界)。これにより「現在の live H1 バーが 1 本確定する」というユーザーの直感に一致する。市場クローズ(週末・祝日)で次のフォーカス TF 境界に取引データが無い場合は **自動的に次の取引バー境界へスキップ**する(土曜から H1 +1本を押すと月曜開場後で最初に確定する H1 バーの終端へジャンプ)。これによりユーザーが見ているメイン足が必ず 1 本進み、土日に置かれて「進めても画面が変わらない」事態を起こさない。フォーカス TF は §5.1.5 参照
- **取得は「最新 N バー保証」**: `chart-stack` は `current_position` 時点での **各 TF 最新 N バー(既定 200 本)を返す**。市場クローズ期間に `current_position` が置かれた場合でも、過去側へ遡ってバーを揃える(時間幅ベースのフェッチで「窓内に該当バー無し」となり空配列を返してはならない)
- **各 TF の最新バー(進行中)は「一つ下の TF」を集約して算出**する(M15 の最新バーは M5 から、H1 の最新バーは集約済み M15 から、… MN1 の最新バーは集約済み W1 から)。これにより `current_position` 以降の未来情報が混入しない

### 5.1.2 時間軸・クロスヘア同期
- **クロスヘア同期**: ある TF でマウスを動かすと、他の TF 上にも同じ時刻の縦線が表示される
- **時間軸パン同期**: ある TF をスクロール・ズームすると、他の TF も対応する時間帯に追従
- 効果: 複数 TF 間の対比(D1 の戻り = H1 の押し目等)を視線移動だけで行える

### 5.1.3 ホイール・ドラッグ操作
縦スクロールとチャートのズームが衝突するため、以下の操作体系とする。

| 操作 | 振る舞い |
| --- | --- |
| ホイール (素) | ページの**縦スクロール**(他 TF を見る) |
| Ctrl + ホイール | そのチャートの**時間軸ズーム** |
| 時間軸のドラッグ | 時間軸のスケール変更 |
| 価格軸のドラッグ | 価格軸のスケール変更 |
| チャート中央のドラッグ | 時間軸のパン |
| タッチ: ピンチ | ズーム |
| タッチ: 1 本指スワイプ | ページスクロール |

- 素のホイール = ページスクロールは Web の慣習(Ctrl + ホイール = ズームは Figma/Google Maps 等でも共通)に一致させ、学習コストを下げる
- 実装では lightweight-charts の `handleScale.mouseWheel` を無効化し、上位 DOM で `ctrlKey` 判定付きの wheel ハンドラを自前実装する

#### 表示本数の維持
- Chart instance は **TF ごとに 1 つだけ永続化** する(銘柄切替で remount しない)。これにより visible range は Chart instance 内の lightweight-charts の state として自然に保持される
- 銘柄切替時は **直前の visible range の `to - from` を width として取得 → 新銘柄の bars に setData → 右端を最新バーに合わせて `setVisibleLogicalRange({from: bars.length-1+RIGHT_OFFSET-width, to: bars.length-1+RIGHT_OFFSET})`** とする。これで「同 TF で同じ表示本数 / 表示密度」がシンボル間で保たれる
- 同 symbol 内の bars 変化(advance / loadMoreHistory)では visible range を触らず、ユーザーの zoom / pan 状態を維持する
- 初回マウント時のみ既定 width(`DEFAULT_VISIBLE_BARS = 150`)で右端揃えする
- `bars.length < width` の TF(broker のヒストリ不足等)では `fitContent()` でフォールバック表示する
- 過去の「`visibleBarsMemory.ts` (TF キーのモジュールスコープ Map)で width を共有する」設計は撤廃した。ライブラリの自動 emit が memory を汚染するパスがあったため

### 5.1.4 その他
- **時刻表示は JST**: 時間軸ラベル、クロスヘア、ツールチップ全て JST 表示(内部データは UTC 保持、表示時のみ変換)
- 表示 TF の選択状態はセッションまたはユーザ設定として保存する(離脱・再入場で復元)

### 5.1.5 フォーカス TF
- 縦積みチャートの中で **「いま操作中心となる TF」= フォーカス TF** を 1 つ持つ。advance(+N 本)・SL/TP 配置・描画作成・エントリー時の `entry_tf` 記録は全てフォーカス TF を基準にする
- **クリックで明示選択**: TF チャート上を 1 度クリック / マウスダウンするとその TF がフォーカスになる。次に別の TF をクリックするまで保持され、マウスホバーでは変わらない(操作中心がぶれないため)
- **ハイライトのみ**: フォーカス TF はチャート枠線・背景の強調表示で示す。スタックの並び順(§5.1.1)は変わらない
- **初期値**: セッション初回マウント時は M5。銘柄切替・TF 表示 ON/OFF・フェーズ遷移ではリセットしない(直前のフォーカスを保持)
- **フェーズ別挙動**:
  - 分析中: 自由切替。クリックで focus、各種操作はフォーカス TF を中心に動く
  - 保有中・振り返り: フォーカスは自由に切替可能(チャートを見比べる / 描画したい TF を選ぶ)。advance も「フォーカス TF の N バー」単位で進む
  - **マーカー(エントリー / 決済の三角)** の表示対象 TF は `Trade.entry_tf`(エントリー時に固定された TF)であり、後からフォーカスを変えても移動しない([§5.5.4](#554-縦マーカーentry_tf-のチャートのみ))

### 5.1.6 下位 TF レンジ背景

- 各 TF pane には「現在表示中の TF 集合のうち **自 TF より 1 つ下に並ぶ TF**」が見ている時間レンジを **半透明の縦帯** で重ねる。マルチ TF で「いま下位足のどこを見ているか」を上位足側から一目で把握するための補助表示
- 「1 つ下」は固定インデックス(M5→M15→H1→…)ではなく **`visibleTfs` 上の隣接関係** で決まる。M5 を非表示にしている間は M15 が最下位となり、M15 pane に帯は出ない(連動データソースが欠落するため)
- 帯の色は §5.3 の TF 色テーブルから **下位 TF の色** を取り、`fillOpacity = 0.08` 程度で適用する。例: 全 TF 表示なら H1 pane に M15 ティール(#56d4dd)、M5 を隠して H1 が直下なら H4 pane に H1 緑(#26a69a)
- 表示中 TF の最下位 pane(M5 が表示中なら M5、隠していれば M15、…)には何も重ねない
- ユーザー設定での ON/OFF は持たない(常時表示)
- **時刻 mapping は logical 経由の純粋関数で行う**: lower の visible logical 範囲を bar 配列 + TF 秒数の線形補間で時刻に変換し(`logicalToTime`)、その時刻を upper の bar 配列 + TF 秒数で再び logical に変換し(`timeToLogical`)、最後に `logicalToCoordinate` で px に落とす一貫経路で帯の左右端を決める。ライブラリの `timeToCoordinate` は使わない(時刻が upper TF のバー境界に一致しないとき null を返す経路があり、フォールバック分岐が増えて TF 間で挙動が分かれる温床になる)
- **両端の edge-snap 規則**: 「両 pane が共に末端を表示している」**AND** 条件のときだけ snap を発火し、それ以外は前項の時刻 mapping に任せる:
  - 右端 snap: lower の visible logical 右端が `>= lower.lastIdx`(rightOffset whitespace 込みの末尾)**かつ** upper の visible logical 右端が `>= upper.lastIdx` のとき → 帯の右端を upper の visible 右端 logical へ snap
  - 左端 snap: lower の visible logical 左端が `<= 0` **かつ** upper の visible logical 左端が `<= 0` のとき → 帯の左端を upper の visible 左端 logical へ snap
- snap が必要な理由は **rightOffset の時間幅不一致** に限定される: lower / upper の rightOffset(各 4 バー)は同じ視覚オフセットだが TF 間隔が違うため時間幅は不一致(M15→H1 なら lower 60min vs upper 240min)。純粋な時刻 mapping だと「両 TF が最新を見ている」ときに band 右端が upper pane の右端まで届かない。両 pane が末端でない通常領域では時刻 mapping で正しく追従するため snap は不要
- **データ範囲の不一致(broker のヒストリ制限)** は snap で吸収しない。upper TF のバー数が lower の時刻レンジに届かない場合、変換後の logical は負値や `lastIdx` 超過になるが、`logicalToCoordinate` の線形外挿(LWC 既定挙動)で px が返るため、帯が pane 外まで伸びる形で自然に処理される。SVG クリップで見た目正しい(過去設計で `timeToCoordinate` が null を返した経路で発生していた「pane 左端から過剰描画」の症状はこの経路では起きない)

### 5.1.7 銘柄切替の応答キャッシュ(クライアント in-memory)

- 銘柄切替の cold load(MT5 から 7 TF 直列フェッチで 700〜1,400 ms 程度)を体感ゼロにするため、`/chart-stack` の **レスポンス全体を `(symbol, current_position, 表示中 TF 集合)` をキーとしたモジュールスコープ Map にキャッシュ** する
- 同タブ内で銘柄を行き来した時の 2 回目以降は **MT5 ラウンドトリップなし**(0 ms 級)で描画される
- `current_position` が advance などで変わると別キーになるためキャッシュミス → 通常通り `/chart-stack` を fetch する
- `reloadStack`(advance 直後の強制再取得)経路は **キャッシュをバイパス**(read / write 共)。advance 直後は `currentPosition` がまだ古い値を指している race があるため、古いキーで誤ヒットしない / 古いキーに新データを誤書込しない
- 永続化はしない(タブを閉じる / ブラウザ刷新で消える)。永続キャッシュは別途検討
- キャッシュ最大 50 エントリ(28 銘柄 × 数 position の往来をカバー)、超過時は LRU で最古を破棄
- 粒度はレスポンス単位(whole-response)。バー単位の per-bar キャッシュは断片化リスクがあるため不採用
- **§5.1.3 の Chart instance 永続化が前提**:銘柄切替で Chart は remount せず、cache hit/miss で bars prop が変わるだけ。visible range は Chart 内で保持されるため、cache を入れても表示位置が壊れない
- **連続切替時の進行中 fetch 中断**: 銘柄を高速で連続切替した際、進行中の `/chart-stack` リクエストを `AbortController` で中断する。frontend は同時に in-flight な fetch を 1 件だけ持ち、新しい銘柄に切り替わった瞬間に古い fetch を abort して新規発行する。これにより MT5 IPC が内部シリアライズで連鎖待機を起こす(古い銘柄の fetch が backend thread pool に滞留して新銘柄の fetch を待たせる)事態を防ぐ

## 5.2 インジケーター(自前実装)
- SMA(単純移動平均、既定 period=20)
- EMA20(指数移動平均、period=20、短期トレンド)
- EMA200(指数移動平均、period=200、長期トレンド)
- RSI(サブパネル表示、period=14)
- 拡張可能な設計で実装(`indicators/registry.ts` にエントリ追加で対応)

### 5.2.1 TF スコープ(ver 1.71 以降)
- **インジケーターは作成時のフォーカス TF にスコープされる**。トグル ON にしたときの `focusedTf` が `IndicatorConfig.timeframe` として記録され、その TF のチャートにのみ描画される
- IndicatorPanel(サイドバー)に表示されるのは **フォーカス TF のインジケーターのみ**。フォーカス TF を切り替えると、パネルの中身もその TF のセットに切り替わる(M5 に SMA(20)、H1 に SMA(50) のように TF ごとに独立した編成が可能)
- 同じ種別 + パラメータでも TF が違えば別インジケーターとして扱われる(`indicatorKey = type-period-tf`)
- 描画ツール(§5.3)と同じ思想: 作成時の TF コンテキストを保持し、混雑した一覧の見づらさを避ける

## 5.3 描画ツール
| ツール | 実装 | デフォルト表示範囲 |
|---|---|---|
| 水平線 | ライブラリ標準 | 全時間足 |
| トレンドライン | ライブラリ標準 | 全時間足 |
| フィボナッチリトレースメント | プラグイン自前実装 | 作成時間足のみ |

- **銘柄別の紐付け**: 描画は作成時の銘柄(`symbol`)に紐付き、`SessionPage` で銘柄を切り替えるとその銘柄の描画だけが表示される。分析中に USDJPY に引いたラインが EURJPY に重ねて表示されることはない。エントリー後は該当銘柄の描画だけが自動的に読み込まれる。
- **作成時間足の自動記録**: 描画対象となったチャートの時間足を `timeframe` として保存する。マルチタイムフレーム分析において「どの時間足を見て引いたライン/カウントか」を後から識別できる。
- **表示対象時間足の個別制御**: `visible_on_timeframes` で表示する時間足を個別指定可能。上表は初期値であり、ユーザーが描画プロパティから変更できる(例: 特定のトレンドラインを上位足でも表示したい等)。
- **時間足の視覚的識別**: 描画の色を作成時の `timeframe` に応じて固定し、一目でどの時間足を見て引いた描画かを識別できるようにする。

  | TF  | 色                         |
  | --- | -------------------------- |
  | M5  | スカイブルー (#58a6ff)      |
  | M15 | ティール (#56d4dd)          |
  | H1  | 緑 (#26a69a)                |
  | H4  | 黄 (#e3b341)                |
  | D1  | オレンジ (#f0883e)          |
  | W1  | 紫 (#d2a8ff)                |
  | MN1 | マゼンタ (#ff79c6)          |

  - ツール種別は形状で判別する(水平線・線分・フィボナッチ)、色は時間足識別に専念する
  - 描画のホバー・選択時には端点付近に TF バッジ(例: `[H1]`)を表示し、色覚配慮の補助情報とする
  - ユーザーが `visible_on_timeframes` を変更して他の時間足に表示している描画も、色・バッジ共に **作成時の時間足** を示す(どの時間足の視点で引かれたかを保持する意図)
- **波動カウント支援**: エリオット波動のカウント UI は描画ツールではなく、§16 Phase 2b "波動ラベリング支援" の専用機能として別途提供する。
- **波動ラベル配置の連続化**: 波動 1→2→3→4→5(推進波)/ A→B→C(補正波)を連続配置するときの操作コストを下げるため、以下の挙動を取る:
  - **Auto-advance**: 波動 N を配置すると自動的に次の波(N+1)を選択した状態に遷移する。`5` を配置したら Idle に戻る(推進波の終端)、`C` を配置したら Idle に戻る(補正波の終端)。連続クリックだけで 1-2-3-4-5 / A-B-C を配置できる
  - **ホットキー**: 描画ツールがアクティブでないときも `1`-`5` / `A`-`C` キーで該当波動の配置モードに直接入る。配置モード中も `1`-`5` / `A`-`C` キーで波動を切替可能(訂正やスキップ用)
  - **`ESC` キー**: 配置モードを抜けて Idle に戻る(既存挙動)
  - **入力欄フォーカス時はホットキーを無効化**(メモ・セッション名欄のテキスト入力を妨げない)。`<input>` / `<textarea>` / `contentEditable` 要素にフォーカスがある間は `1`-`5` / `A`-`C` を捕捉しない
- **バー範囲外クリック**: 最終バーより右(whitespace)・最初のバーより左の位置でも描画が可能。lightweight-charts の `coordinateToTime` が範囲外で null を返すため、frontend 側で `coordinateToLogical` の値と TF 間隔(`TIMEFRAME_MINUTES`)から線形外挿した時刻を生成する。これによりトレンドラインやフィボナッチの頂点を未来側(右の余白)に置く運用が可能になる。MN1 は月毎日数差を 30 日近似で扱う。

## 5.4 経済指標表示
- **表示対象**: 重要度 **星3のみ**(設定で星2以上への拡張も可能な設計)
- **通貨フィルタ**: 表示中チャートの通貨ペアに関連する通貨 + USD(基軸通貨)
- **表示スタイル**:
  - **縦線マーカー**: 発表時刻に縦線(通貨別に色分け、ホバーで詳細表示)
  - **シェーディング**: 発表前後の時間帯(例: 前5分〜後30分)を背景色で塗り、ボラ上昇期間を可視化
- **時間軸上の情報公開ポリシー**(実戦で得られる情報と同じ扱い):
  - **未来の指標も表示**: 発表予定時刻、指標名、通貨、重要度、**予想値**、前回値はセッション現在時刻より未来でも表示(実戦で事前に知れる情報)
  - **実測値のみ発表時刻到達後に表示**: サプライズ度も実測値公開後に算出・表示
  - 縦線・シェーディングは未来分も含めて常時表示(次の指標までの残り時間を意識できる)
- **ホバー情報**: 指標名、通貨、発表時刻、実測値/予想値/前回値、サプライズ度(実測 vs 予想の乖離)

## 5.5 エントリー / 決済情報の表示

エントリー後・決済後の Trade 情報をチャート上に視覚化する。判断時点(分析中)に置いた SL/TP draft の表示([§7.4](./07-memo.md#74-エントリー時の必須項目))も同じ仕組みを使う。

| **振り返り**(見送り) | — | — | — | — | — | — |

### 5.5.3 横線(全 TF)
- エントリー後・決済後は `Trade.entry_price / sl / tp / exit_price` の値を全 TF の priceLine に注入する
- 描画オブジェクト由来の水平線(§5.3)と同じ priceLine 機構を使う
- ラベルに価格を併記(例: `Entry @ 150.123`、`SL @ 149.800`)
- 銘柄が確定済の保有中・振り返りでは銘柄切替不可([§6.1](./06-session-screen.md#61-画面構成))のため、表示銘柄と Trade 銘柄は常に一致する

### 5.5.4 縦マーカー(`entry_tf` のチャートのみ)
- エントリー / 決済時刻に三角マーカーを配置(lightweight-charts 標準 `setMarkers` API)
- マーカーが意味を持つのは **`Trade.entry_tf`(エントリー時に固定されたフォーカス TF)** のチャートだけ。他 TF はバー時刻と Trade 時刻が一致せずマーカーがズレるため非表示
- 後からフォーカス TF を変えてもマーカーは移動しない(エントリー時の TF 視点を保持する設計)
- マーカー位置は `entry_time` / `exit_time` を **`entry_tf` の bar 開始時刻に丸めた値**を使う
- shape: BUY エントリー = `arrowUp` (belowBar)、SELL エントリー = `arrowDown` (aboveBar)、決済はその逆向き

## 5.6 描画の保存
セッションごとに描画オブジェクトを保存し、後から視覚的に振り返り可能。

描画には以下のメタデータを保持する:
- `symbol`: 描画を行った銘柄(必須)。銘柄切替時に該当銘柄の描画のみ表示するために使う。
- `timeframe`: 描画を行ったチャートの時間足(必須)。判断根拠の時間軸コンテキストとしてAI分析にも活用する。
- `visible_on_timeframes`: 表示する時間足リスト(JSON 配列)。種類ごとの既定値は §5.3 参照、ユーザーが個別に変更可能。
