## 4.1 通常セッション(統合フロー・1 画面完結)

画面分割はしない。「銘柄選定」「トレーニング」と段階分けするのではなく、**1 つのセッション画面** の中で、分析 → エントリー(= そこで初めて銘柄が確定する) → 保有 → 決済 → 振り返り が流れる。

```
[Phase 1: 時間選定]
1. 時間フィルタ設定(セッション・曜日・期間など)
2. システムがランダムな日時を提示してセッション開始(銘柄はまだ決まっていない)

[Phase 2: 分析とエントリー判断(1 画面内で完結)]
3. 銘柄リストから気になる銘柄を選んで詳細チャート(マルチタイムフレーム)で観察
4. 描画ツール・インジケーターで分析(サポレジ・トレンドライン・波動カウント 等)
5. メモパネル(§7.3、M キー or ボタン)を開閉しながら、銘柄別メモに個別の気づき、
   横断メモに通貨強弱・銘柄比較・相場観・シナリオを書く
6. 現時点がエントリーに適さなければ ▶ で足を進めて待つ。環境が変わったら分析をやり直す
7. 比較のために別銘柄に切り替えてもよい(描画は銘柄別に保持され、切替で自動的に該当銘柄の描画に切り替わる)
8. ★ で気になる銘柄をブックマーク(任意、銘柄別メモの入れ物として機能)
9. 条件が揃った銘柄で「方向・価格・SL・TP・スタイル」を入力してエントリー
   または、優位性なしと判断したら「見送り」
   ※ エントリーアクションそのものが「この銘柄でこのセッションをやる」という選定確定
   ※ 必要な思考(根拠・シナリオ・スタイル選定理由)は銘柄別メモ・横断メモに事前記入済みの前提

[Phase 3: 保有中(エントリーした場合のみ)]
10. 銘柄は Phase 2 のエントリーで確定。以降は切替できない
11. ▶ で足を進めて TP/SL 到達 or 裁量決済まで進む。必要なら描画を追加、メモに気づき追記
12. 決済(自動 or 裁量)

[Phase 4: 振り返り]
13. 決済後の続き観察(追加 N 本の OHLC チャート + MFE・MAE・実損益 R を R 単位で確認、[§9.5](./09-post-review.md#95-エントリー結果の事後確認))
14. 見送り時は事後チャートと §9.3 自動判定指標を PostReviewPanel で確認
15. 横断メモに振り返り追記(後日追記可、§7.7 / [principles/no-aggregation.md#103-セッションのライフサイクル](./principles/no-aggregation.md#103-セッションのライフサイクル))
16. 完了したら「セッションを閉じる」で DB から破棄([principles/no-aggregation.md#103-セッションのライフサイクル](./principles/no-aggregation.md#103-セッションのライフサイクル))
```

## 4.2 セッション管理
- 時間制限なし
- 途中保存・再開機能あり
- **1セッション1エントリー完結**(trainer は単一判断の訓練に特化)
- 複数ポジション管理(資金配分・優先順位判断・ヘッジ・段階エントリー)は trainer の対象外。これらは実戦環境(journal)で経験として学ぶ方針
- trainer で身につけるのは「1つの局面に対するエントリー・決済判断の質」。実戦で必要な「同時に複数のポジションをどう捌くか」は journal 側で扱う
- **Phase の行き来を許容する**: §4.1 のフローは概念的な順序であり、実際の訓練では Phase 2 の中で「銘柄切替 ↔ 分析 ↔ 足送り ↔ メモ」を自由に往復できる。Phase 3 以降(エントリー後)は銘柄が確定するが、描画・メモ追記は継続可能。
- **画面遷移は一度もない**: 分析・エントリー・保有・決済・振り返りはすべて同じ `SessionPage` 内で行い、セッションの状態(アクティブ Trade の有無 / skip 済か)からフェーズを導出して UI 要素を出し分ける。
