# 対案分析(Counter-View)機能の追加

ステータス: **draft / 着手前**(2026-05-05 議論メモ)
仕様化 / 実装は別セッションで実施する。

## 背景・動機

ユーザーから「エントリー前に現在の相場分析の意見を聞きたい」との要望。
深掘りした結果、求めているのは**思考の外注ではなく対立仮説の強制**であることが
明確化:

> 自分で考えた相場の方向感とは別の意見があれば参考にしたい。
> 特にエリオット波動の数え方や値幅は複数の考え方が存在するので、
> 別の考え方について知りたい。

エリオット波動・値幅予測は主観性が高く、自分のカウント / 解釈に固執して
代替仮説を見落とすリスクが大きい。「赤チーム」用途で AI に対案を出させる
ことで、自分の見立ての盲点を意識させる狙い。

既存の `AiAnalysisPanel` (§11) は **エントリー後 / 振り返り時** の観察者役で、
今回の機能は **エントリー前の対案提示** で目的が異なる。コードもパネルも
分離する。

## 確定済みの設計判断

| 軸 | 決定 |
|---|---|
| 対象 TF | **全 TF (M5〜MN1)** |
| 波動描画 (§5.3 wave_label) を AI 入力に含めるか | **含める**(他の line / trendline / fibonacci は渡さない) |
| メモ不足時の扱い | **AI にメモの不足点を指摘させる**(UI 側で blocking しない) |
| UI 配置 | **後ほど検討**(§11 隣 / 別タブ / モーダル等) |

### コア設計判断(§11.9 ガードレール拡張)

1. **ユーザーのメモを必須入力に含める**
   - メモを書かせる前提を AI 出力経由で促す(blocking はしない、UI で書かせる強制も
     しない。AI が「メモが薄い」と指摘する形で fb をかける)
   - これがないと「自分で考える前に AI に聞く」状態になり §11.9 観察者ガードレール違反

2. **AI への入力**
   - チャート画像(全 TF、現在 cursor 時点)
   - ユーザーの分析メモ(現在 cursor の)
   - **§5.3 wave_label Drawing**(各 TF の波動ラベル: 名前 / 時刻 / 価格)
   - **送らない**: SL/TP draft、過去 Trade、損益、ラベル付き結果。「結果が分かる材料」一切なし

3. **AI への出力指示**
   - レポート構造は **固定 4 セクション**(下記)
   - **方向(BUY/SELL/様子見)の推奨は禁止**(プロンプトで明示的に縛る)
   - ユーザーのカウントを「正しい / 間違っている」と判定しない(エリオットはどれもあり得る)

## レポート構造(案、未確定)

提案した 4 セクション構成。仕様化時に最終確定する:

1. **あなたのメモの読み取り**
   - ユーザーのメモから読み取れる方向観 / カウント / 値幅観の要約
   - メモが薄い / 抜けがある場合はここで指摘
     (例: 「上位 TF への言及がありません」「カウントが 1 つしか書かれていません」)
   - 判定はしない、要約と不足指摘のみ

2. **別カウント候補(最大 2 つ)**
   - 各候補: 別のラベリング案 / 採用根拠(どの上位 TF 文脈・どの対称性)/
     採用した場合の次の注目価格・時刻
   - 「方向の意見」ではなく「カウントの解釈」のみ

3. **別の値幅観(2〜3 個)**
   - フィボ拡張 / チャネル / 前波対称性 / 上位 TF サポレジ
   - 価格レベルのみ、推奨方向は付けない

4. **対案を念頭に置いて自問するべきこと(3〜5 個)**
   - §11 自問セクションと同じ形式
